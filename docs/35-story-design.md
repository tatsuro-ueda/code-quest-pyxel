# ストーリー設計

story-concepts.md のコンセプトを実装に落とし込んだ全テキスト仕様。

---

## 1. プロフェッサーのセリフ変化

城訪問時のセリフを `player.maxZoneReached` で分岐させる。

### 序盤（maxZoneReached ≤ 0）
> プロフェッサー「バグの魔王グリッチがブロックを暴走させた。きみが理解して仲間に戻すのだ！町に立ち寄り装備を整えよう。」

### 中盤（maxZoneReached 1〜2）
> プロフェッサー「なぜお前だけが気づくのか、考えたことはあるか？」

### 終盤（maxZoneReached ≥ 3）
> プロフェッサー「それでもなお、理解し続けるのか？」

※HP/MP全回復・毒治療はすべての段階で維持。

## 2. NPCセリフ（全差し替え）

各町3人、計9人。ゾーンテーマに沿った内容に変更。

### はじめの村（Zone 0 テーマ：違和感）— 世間の声

町の人々は「気にしなくていい」側の人間。プレイヤーは疎外感を覚える。

1. 「ものごとには順番があるけど、少しズレたって気にならないだろ？」
2. 「そんなに気にしてたら何もできないよ。大体あってればいいんだよ」
3. 「順番？ 動けばいいんだよ、動けば」

### ロジックタウン（Zone 1 テーマ：反発）— 世間の声

1. 「同じことの繰り返し？ そういうもんだろ、仕事って」
2. 「考えすぎだよ。回ってるならそれでいいじゃないか」
3. 「止める？ なんで動いてるものを止めるんだ？」

### アルゴリズムの街（Zone 2 テーマ：自覚）— 世間の声

1. 「場合によるとか言い出すと、話が進まないんだよな」
2. 「数えなくたって、なんとなく分かるだろ？」
3. 「そこまで見えなくても、生きていけるぞ」

### 4.1 戦闘開始ログ

### 4.2 プレイヤー通常攻撃ログ

フレーバーテキスト付き。ダメージ数値は残す。5パターンをランダム。

※5分クリア想定で全体の戦闘回数が15〜20回のため、同じパターンの繰り返し感は薄い。

- `順番を見直した。○○に○のダメージ！`
- `動きを観察した。○○に○のダメージ！`
- `構造を読み取った。○○に○のダメージ！`
- `流れをたどった。○○に○のダメージ！`
- `原因に迫った。○○に○のダメージ！`

| スキル | テキスト |
|--------|---------|

## 5. ボス戦専用ログ（HPフェーズ制）

### ダメージ試算

| 想定レベル | 武器 | 平均ダメージ | 倒すまでのターン数 |
|-----------|------|-------------|-------------------|
| Lv15 | テキストエディタ(18) | ~43 | ~12 |
| Lv18 | コードエディタ(30) | ~63 | ~8 |
| Lv20 | デバッガー(45) | ~84 | ~6 |
| Lv25 | コンパイラ(65) | ~120 | ~5 |

想定プレイヤーはLv18〜22で到達。6〜10ターンの間にセリフが出る設計。

### フェーズ定義（6段階 + 撃破）

`battleState.bossPhase` で管理。敵ターン終了後にHP割合を判定し、未表示フェーズがあれば `battleState.msg` を上書き（通常の敵行動ログの代わりにセリフを表示）。

| # | HP条件 | セリフ |
|---|--------|--------|
| 開始 | 戦闘開始時 | `グリッチが現れた。理解を拒んでいる。` |
| 1 | HP < 80% | `グリッチ「わたしは、まちがえたかったわけじゃない！」「ちゃんとできれば、ほめてもらえると思ってた」` |
| 2 | HP < 60% | `グリッチ「パパは、むずかしいことをたくさん教えてくれた」「わからないままでも、うなずけば笑ってくれた！」` |
| 3 | HP < 40% | `グリッチ「こわせって言われたわけじゃない」「ただ、まもろうとしただけ！」` |
| 4 | HP < 20% | `グリッチ「でも、まもりかたがわからなかった！」「こわしたら、もう離れていかない気がした」` |
| 5 | HP < 8% | `グリッチ「おまえは、見るんだな？」「バグじゃなくて、わたしを」` |
| 撃破 | HP = 0 | `グリッチを理解した。グリッチ「わたしのことを、わかってほしい」「それでも、ここにいたかったことも……！」` |

### 実装方式

`battleState.bossPhase` (初期値0) をインクリメントして管理。
セリフは `b.msg` に設定し `enemyAction` フェーズで表示。
→ 通常の敵ターンメッセージと同じ表示方式。テスト容易性を維持。
→ `showMsg()` は使わない（バトル状態とメッセージの二重状態を避ける）。

撃破セリフは `processVictory` 内で `b.msg` を差し替え。
通常の「○○を理解した！○EXPと○Cを手に入れた！」の代わりにボス専用テキストを設定。

## 7. エンディング

### 遷移条件の変更

旧：`bossDefeated && gameState==='map' && !msgCurrent` → 即エンディング。

新：ダンジョン脱出時（`player.inDungeon=false` に切り替わるタイミング）かつ `bossDefeated` のとき、ダンジョン内の敵出現を無効化し、脱出メッセージ表示後にエンディングへ遷移。

具体的には：
1. ボス撃破後、`bossDefeated=true` になる
2. ダンジョン内を歩いて入口(y=0)に向かう。この間エンカウントなし（`bossDefeated` で判定）
3. 脱出メッセージ表示
4. メッセージ消化後にエンディングへ遷移

### エンディングテキスト（lines配列差し替え）

```
🎉 デバッグ完了！ 🎉

きみは理解した。

世界は、完全ではない。
だからこそ、見えてしまう者が必要だ。

きみはもう見習いではない。
「見えてしまう側の人間」として、
この世界に立っている。

- FIN -

Enterキーでタイトルに戻る
```

## 10. 実装変更の概要

### 新規フラグ（playerオブジェクトに追加、セーブ対象）
- `maxZoneReached` — 到達最大ゾーン（初期値0）。`updateBGM()` 内で更新
- `landmarkTreeSeen` — 世界樹メッセージ表示済み（初期値false）
- `landmarkTowerSeen` — 通信塔メッセージ表示済み（初期値false）

### 新規フラグ（battleStateに追加、セーブ対象外）
- `bossPhase` — ボス戦フェーズカウンター（初期値0）

### 敵データ変更
- `category` フィールド追加（`'sequential'|'loop'|'condition'|'variable'|'composite'|'boss'`）

### バトルフェーズ追加
- `escaped` — 逃走成功フェーズ（`victory` と分離）

### セーブバージョン
- `saveVersion` を 2 → 3 に変更
- ロード時に `maxZoneReached`, `landmarkTreeSeen`, `landmarkTowerSeen` が未定義なら初期値で補完

### エンディング遷移変更
- 旧：`bossDefeated && gameState==='map' && !msgCurrent` → 即エンディング
- 新：ダンジョン脱出処理内で `bossDefeated` 判定 → 脱出メッセージ後にエンディング遷移
- ダンジョン内 `checkEncounter()` で `bossDefeated` 時はエンカウント無効

## 11. 裏ボス「プロフェッサー」

エンディング後に到達可能な裏ボス。
プロフェッサーは「支配者」「権力」「全体主義」の象徴として描く。
語り口は演説調を基調にしつつ、断言の一部にだけ強い句読点を置く。
