# ビジュアル設計

本ドキュメントは `visual-requirements.md` の8手法をブロッククエストの実装に落とし込む設計書である。
現行コードの描画層（`drawTile`, `drawMap`, `drawHero`, `drawBattle`, `drawTitle` 等）に対して
**何を・どう変えるか**を定義する。

> 制約: 単一 `index.html`、Canvas 2D API のみ、外部画像/スプライトシート不使用。

---

## 設計方針

| レイヤー | 対応する要件手法 | 本ゲームでの実装方針 |
|----------|-----------------|--------------------|
| 見た目を良くする | ①タイル変化 ②高低差 ④色/ライティング | タイルバリエーション + ゾーン別パレット |
| 迷わせない | ③ランドマーク ⑥道デザイン ⑦密度制御 | 構造物差別化 + 道タイル + 疎密設計 |
| 世界に入り込ませる | ⑤アニメーション ⑧触れそう感 | 水/草アニメ + NPC歩行 + 調べられるオブジェクト |

## 1. タイルシステム改善

### 1.1 現状

```
drawTile(x, y, type) — switch文で11タイル種を描画
各タイルは固定色 + 固定装飾（草: 3本の草ブレード、水: 2本の波線 等）
```

全タイルが座標によらず同一見た目 → 単調。

### 1.2 タイルバリエーション（手法①）

**方式**: 座標ベースの疑似乱数でバリエーション選択（`tileVariant(x, y)`）。
`Math.random()` は使わず座標のハッシュで決定論的に選ぶ（再描画で安定）。

```js
function tileHash(x, y) {
  return ((x * 7841 + y * 5923) & 0xffff) / 0xffff; // 0..1
}
```

| タイル | バリエーション数 | 差分内容 |
|--------|:---:|------|
| GRASS | 4 | 草ブレードの本数(1-4)、位置オフセット、花(黄/白)の有無 |
| WATER | 2 | 波線パターンA/B（位置ずらし） |
| TREE | 3 | 葉の大きさ(小/中/大)、色味(#226622/#1a7a1a/#2d5a2d) |
| SAND | 3 | 砂粒の配置パターン、色(#ddcc88/#d4c480/#e0d090) |
| FLOOR | 2 | 石板の向き（チェッカーA/B） |
| WALL | 2 | レンガ模様の横線位置 |

**変更箇所**: `drawTile()` 内の各 case に `tileHash(x,y)` 分岐を追加。

### 1.3 端タイル処理（地形境界）

隣接タイルとの差異で端を表現する。完全オートタイルではなく簡易方式：

- **水際**: 草が水に隣接する辺に波打ち際の薄青ライン(2px)を描画
- **森の影**: 木タイルの南側の草タイルに薄い影(rgba(0,0,0,0.15))を描画

```js
function drawTileEdge(vx, vy, type, neighbors) {
  // neighbors: {n,s,e,w} — 隣接タイル種
  if (type === TILE_GRASS && neighbors.n === TILE_WATER) { /* 波打ち際 */ }
  if (type === TILE_GRASS && neighbors.n === TILE_TREE) { /* 木の影 */ }
}
```

**変更箇所**: `drawMap()` のタイル描画ループ内、`drawTile()` 呼び出し後に `drawTileEdge()` を追加。

## 2. ゾーン別パレット（手法④ 色/ライティング）

### 2.1 現状

草は全域 `#33aa33` 固定。4ゾーン＋ダンジョンの区別が色では分からない。

### 2.2 ゾーンパレット定義

プレイヤー座標から `getEnemyZone()` で決まるゾーンに連動してベースカラーをシフトする。

| ゾーン | エリア名 | 草ベース | 地面アクセント | 雰囲気 |
|:---:|----------|----------|---------------|--------|
| 0 | 城〜はじめの村周辺 | `#33aa33`（明緑） | `#44bb44` | 安心・平和 |
| 1 | ループ地帯 | `#2d9930`（やや暗緑） | `#3daa3d` | 少し深い森 |
| 2 | 条件分岐エリア | `#287028`（暗緑） | `#386838` | 不穏・薄暗い |
| 3 | 変数/クローンエリア | `#445533`（枯草混じり） | `#556644` | 荒廃感 |
| 4 | ダンジョン | — | — | 別マップ（後述） |

**実装方式**: `ZONE_PALETTE[zone]` テーブルを定義。`drawTile()` で `TILE_GRASS` の場合、
現在ゾーンに応じたベースカラーを参照する。

```js
const ZONE_PALETTE = [
  { grass: '#33aa33', accent: '#44bb44', water: '#2255cc' },
  { grass: '#2d9930', accent: '#3daa3d', water: '#224499' },
  { grass: '#287028', accent: '#386838', water: '#1a3388' },
  { grass: '#445533', accent: '#556644', water: '#2a2a66' },
];
```

**ゾーン境界**: 隣接ゾーンのパレットを補間はしない（DQ的にくっきり切り替わる）。
ただしマップ上に地形（山脈・川）がゾーン境界を暗示する配置になっている。

### 2.3 ダンジョン配色

現行: 床 `#554433` / 壁 `#333333`。変更なし（十分な雰囲気がある）。
ボス部屋（BOSS_POS周辺3x3）の床を `#442233` に変えて不穏さを追加。

## 3. ランドマーク差別化（手法③）

### 3.1 現状

町(TILE_TOWN)と城(TILE_CASTLE)は 32x32 で描画。小さくて見つけにくい。

### 3.2 改善

**町/城/洞窟の目立たせ方**:

| 構造物 | 追加描画 |
|--------|----------|
| CASTLE | 旗アニメ（頂上に2pxの旗が `animTimer` で揺れる） |
| TOWN | 煙アニメ（屋根から1px の煙パーティクルが上昇、`animTimer` 連動） |
| CAVE | 入口の闇が明滅（`rgba` の alpha を `0.8+sin(t)*0.2` で揺らす） |

**`animTimer`** は既にゲームループで更新されている変数を流用する。

**変更箇所**: `drawTile()` 内の CASTLE/TOWN/CAVE case に追加描画。
`animTimer` を `drawTile()` の引数に追加するか、グローバル参照で実装。

### 3.3 町名表示

プレイヤーが町/城タイルの隣接4マス以内に入ったとき、タイル上方に町名をフロート表示。

```js
// drawMap() 内、ヒーロー描画後
TOWNS.forEach(t => {
  if (Math.abs(player.x - t.x) + Math.abs(player.y - t.y) <= 2) {
    drawText(t.name, (t.x - camX) * TILE, (t.y - camY) * TILE - 12, 11, '#ffee88');
  }
});
```

## 4. 水アニメーション（手法⑤）

### 4.1 現状

水タイルは静的な2本の横線。

### 4.2 改善

波線の y オフセットを `animTimer` で周期的にシフトする。

```js
case TILE_WATER:
  ctx.fillStyle = palette.water;
  ctx.fillRect(sx, sy, TILE, TILE);
  ctx.fillStyle = lighten(palette.water);
  const waveOff = Math.sin(animTimer * 0.003 + x * 0.5) * 2;
  ctx.fillRect(sx + 4, sy + 10 + waveOff, 12, 3);
  ctx.fillRect(sx + 16, sy + 20 - waveOff, 12, 3);
  break;
```

コスト: `Math.sin` は画面内タイル数(最大 ~200)× 60fps で呼ばれるが問題ない。

## 5. ヒーロー歩行アニメ（手法⑤）

### 5.1 現状

`drawHero()` は固定ポーズ。移動中も静止画。

### 5.2 改善

歩行中（`keys` にいずれかの方向が入っている間）は足の位置を2フレームで切り替える。

```
フレーム0: 左足前（現状の足位置）
フレーム1: 右足前（左右の矩形位置を swap）
```

**歩行フレーム判定**: `Math.floor(animTimer / 200) % 2` で 0/1 を切り替え。

**変更箇所**: `drawHero(sx, sy)` 内の Legs セクション。

### 5.3 向き

現状ヒーローは常に正面向き。4方向対応は描画コスト大（ピクセルアートで4パターン）のため、
**向きインジケータ**で代替：移動方向に合わせて目の位置を 1px ずらす。

| 入力 | 左目 dx | 右目 dx | 左目 dy | 右目 dy |
|------|:---:|:---:|:---:|:---:|
| 上 | 0 | 0 | -1 | -1 |
| 下 | 0 | 0 | +1 | +1 |
| 左 | -1 | -1 | 0 | 0 |
| 右 | +1 | +1 | 0 | 0 |

**新規グローバル**: `playerDir` ('up'/'down'/'left'/'right'、デフォルト 'down')。
マップ移動処理で更新。`resetGameState()` でリセット。

## 6. 道デザイン（手法⑥）

### 6.1 現状

マップ生成で城への道を `TILE_GRASS` で確保しているが、見た目は草原と同じ。

### 6.2 道タイル追加

新タイル `TILE_PATH = 11` を追加。通行可能（`canWalk` に追加）、エンカウント率 0%。

描画: 草ベース + 中央に土色(#aa9966)の帯(幅12px)。

```js
case TILE_PATH:
  ctx.fillStyle = palette.grass;
  ctx.fillRect(sx, sy, TILE, TILE);
  ctx.fillStyle = '#aa9966';
  ctx.fillRect(sx + 10, sy, 12, TILE); // 縦道
  break;
```

**マップ生成での配置**:
- 城→はじめの村を結ぶ主要道
- 各町間を緩やかに結ぶルート
- 橋の前後2マスに道タイルを配置（橋の存在を暗示）

道の方向（縦/横）は隣接 PATH タイルから判定して描画向きを変える。

```js
// 道の向き判定
const pathH = neighbors.e === TILE_PATH || neighbors.w === TILE_PATH;
const pathV = neighbors.n === TILE_PATH || neighbors.s === TILE_PATH;
// 交差点は十字、行き止まりは方向に合わせる
```

## 7. 密度コントロール（手法⑦）

### 7.1 現状

マップ上に地形はあるが、城周辺と最終エリアで視覚的密度の差がない。

### 7.2 改善

| ゾーン | 密度方針 | 実装 |
|:---:|----------|------|
| 0（序盤） | 疎：広い草原、木は少ない | forest()の半径を小さく、密度0.3 |
| 1 | 中：森が増える | 密度0.5 |
| 2 | 密：森+山が多い | 密度0.7、山も追加 |
| 3（終盤） | 密＋荒廃 | 砂タイル混在、森は枯れ色 |

マップ生成関数 `forest(cx,cy,r)` の `Math.random()<0.6` の閾値をゾーン別に変える。

## 8. バトル画面改善

### 8.1 現状

- 屋外: 上半分に空グラデーション + 下半分に草
- ダンジョン: 上下で紫暗色の2分割
- 敵スプライトは `drawEnemySprite()` でプロシージャル描画

### 8.2 ゾーン別バトル背景（手法④）

`drawBattle()` 内のバトル背景にゾーンパレットを反映する。

| ゾーン | 空の色 | 地面の色 |
|:---:|--------|----------|
| 0 | `#4488ff` → `#88bbff` | `#33aa33` |
| 1 | `#3377dd` → `#77aadd` | `#2d9930` |
| 2 | `#335599` → `#556688` | `#287028` |
| 3 | `#333355` → `#444455` | `#445533` |
| 4 (ダンジョン) | 変更なし | 変更なし |

### 8.3 敵被ダメージ演出改善

現行: `shakeTimer > 0` で敵座標にランダムオフセット。

追加:
- 被ダメージ時に敵スプライトを一瞬白くフラッシュ（`ctx.globalCompositeOperation = 'lighter'` で白矩形を重ねる）
- 倒した瞬間にスプライトを上から消していくワイプ演出（`clipRect` で描画領域を縮小、`defeatAnimTimer` で制御）

## 10. ステータスバー改善

### 10.1 現状

`drawMap()` 内で画面上部40pxにDQウィンドウ + テキスト1行。

### 10.2 改善

- HP/MPにバー表示を追加（テキストの下に細いゲージ、HP緑/赤切替、MP青）
- バグ汚染(poisoned)時にステータスバー全体に紫のパルスオーバーレイ

```js
// HPバー
const hpRatio = player.hp / player.maxHp;
ctx.fillStyle = '#333'; ctx.fillRect(barX, barY, barW, 4);
ctx.fillStyle = hpRatio > 0.25 ? '#44dd44' : '#dd4444';
ctx.fillRect(barX, barY, barW * hpRatio, 4);
```

### 11.1 現状
