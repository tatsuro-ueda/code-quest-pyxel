# 実験案 v5：カスタマージャーニー ── 価値階層軸

> 実験ラベル：**v5 / 価値階層軸**
> 作成日：2026-04-25
> 視点：1 つのジャーニーを「**4 つの価値階層を順に立ち上げていくプロセス**」として描く。同じ瞬間が L1 → L2 → L3 → L4 の価値を連続的に提供する。
> 根拠：[`experimental-customer-jobs-v5.md`](./experimental-customer-jobs-v5.md)

---

## 凡例（v5 固有）

- **subgraph の構造**：`L1 機能` → `L2 情緒` → `L3 社会` → `L4 人生変容`
- 各層内のノードはその層の体験を描く
- ノード形式：`[（主体）絵文字 文（タッチポイント）]`
- ジャーニー末尾に「**この瞬間に立ち上がった価値の層**」を 1 行

---

## 代表ジャーニー（4 層構造での書き直し）

---

### CJ08-v5: 敵が強すぎる

**価値階層的な意味**：「敵を弱くする」という機能（L1）が、「集中の連続」（L2）→「友達と笑える」（L3）→「家族で一緒に直した記憶」（L4）まで貫通する瞬間。

```mermaid
flowchart LR
    subgraph L1["L1: 機能的価値"]
        A1[（子）💢「敵強すぎ！」（ゲーム中）]
        A1 --> A2[（親）👆AIに「スライムのHP 50→30」（AI入力画面）]
        A2 --> A3[（ビルド）👀ヘッドレス検証通過（ビルドログ）]
    end
    subgraph L2["L2: 情緒的価値"]
        A4[（子）👀開発版で 2 かい／本番で 4 かい体感差（ゲーム中）]
        A4 --> A5[（子）❤️自分の判断で「開発版のほう！」承認（選択ページ）]
        A5 --> A6[（子）❤️続きを楽しめる集中の連続（ゲーム中）]
    end
    subgraph L3["L3: 社会的価値"]
        A7[（子）👆友達に「いいバランスになったから遊んで！」とURL再送（LINE）]
        A7 --> A8[（友達）❤️「楽しい！」とフィードバック（スマホブラウザ）]
    end
    subgraph L4["L4: 人生変容価値"]
        A9[（家族）❤️「あの日スライム弱くしてよかったね」と数ヶ月後に振り返る（履歴UI）]
    end

    A3 --> A4
    A6 --> A7
    A8 --> A9
```

> **この瞬間に立ち上がった価値**：L1（HP 変更）→ L2（集中の連続、自己決定）→ L3（友達に共有）→ L4（家族の記憶）。**機能の 1 操作が 4 層全てを貫通**する。

### CJ22-v5: 友達のフィードバックを反映する

**価値階層的な意味**：友達のフィードバックを反映する 1 つの行為が、L1 から L4 まで貫通する。

```mermaid
flowchart LR
    subgraph L1["L1: 機能的価値（修正と配信）"]
        A1[（友達）💡「ここ難しい」（スマホブラウザ）]
        A1 --> A2[（親）👆AIに翻訳（AI入力画面）]
        A2 --> A3[（ビルド）👀検証通過 → 開発版に反映（承認キュー）]
    end
    subgraph L2["L2: 情緒的価値（子の主体性）"]
        A4[（子）👆「直して！」と自分で判断（対面）]
        A4 --> A5[（子）👀開発版を遊び比べて承認（選択ページ）]
    end
    subgraph L3["L3: 社会的価値（友達との往復）"]
        A6[（子）👆URL を友達に再送（LINE）]
        A6 --> A7[（友達）❤️「お、いい感じ！」（スマホブラウザ）]
    end
    subgraph L4["L4: 人生変容価値（家族 identity）"]
        A8[（家族）❤️友達のフィードバックを家族のサイクルに取り込む流れが習慣化（対面）]
    end

    A3 --> A4
    A5 --> A6
    A7 --> A8
```

> **この瞬間に立ち上がった価値**：友達フィードバックの反映は、家族の「外と関わる仕方」そのものを形成する。L4 に届く。

### CJ31-v5: 子どもが変更を承認する

**価値階層的な意味**：「承認する」という単純な行為が、子の自己効力感（L4 の中核）を作る。

```mermaid
flowchart LR
    subgraph L1["L1: 機能的価値"]
        A1[（親）👆AIに依頼 → 開発版に反映（承認キュー）]
    end
    subgraph L2["L2: 情緒的価値（体感での判断）"]
        A2[（子）👀開発版／本番を遊び比べ（ゲーム中）]
        A2 --> A3[（子）❤️体感差で承認 → 自分で決めた（選択ページ）]
    end
    subgraph L3["L3: 社会的価値"]
        A4[（子）👆「これでバランスいいから友達にも見せられる」（対面）]
    end
    subgraph L4["L4: 人生変容価値（自己効力感）"]
        A5[（子）❤️「自分は意思決定できる人だ」という体験が積み上がる（履歴UI）]
    end

    A1 --> A2
    A3 --> A4
    A4 --> A5
```

> **この瞬間に立ち上がった価値**：「承認する」が**繰り返される**ことで、L4 の自己効力感が育つ。1 回の承認だけでは L2 で終わる。

### CJ34-v5: 承認後に「やっぱり」となる

**価値階層的な意味**：「戻す」という一見ネガティブな行為が、L4 の家族 identity に積み上がる。

```mermaid
flowchart LR
    subgraph L1["L1: 機能的価値（戻す機構）"]
        A1[（親）👆「もどして」依頼を承認キューに登録（承認キュー）]
    end
    subgraph L2["L2: 情緒的価値（後悔を可逆に）"]
        A2[（子）❤️自分で承認して戻す（選択ページ）]
        A2 --> A3[（子）❤️戻すのが恥ずかしくない感覚（対面）]
    end
    subgraph L3["L3: 社会的価値"]
        A4[（家族）👆「やっぱりこっちでいくね」と友達にも説明できる（対面）]
    end
    subgraph L4["L4: 人生変容価値（後悔を進化として扱う）"]
        A5[（家族）❤️後悔・修正・再判断のプロセスが家族の判断様式として定着（履歴UI）]
    end

    A1 --> A2
    A3 --> A4
    A4 --> A5
```

> **この瞬間に立ち上がった価値**：「戻す」を恥ずかしくない経験として積むことで、家族全体の判断文化が変わる。

### CJ30-v5: エンディングを自分たちで書く

**価値階層的な意味**：プロダクト最大のクライマックス。L1 から L4 までフルに作動する。

```mermaid
flowchart LR
    subgraph L1["L1: 機能的価値"]
        A1[（親子）👆セリフを考える → AIが実装（AI入力画面）]
        A1 --> A2[（ビルド）👀検証通過（ビルドログ）]
    end
    subgraph L2["L2: 情緒的価値（達成感）"]
        A3[（親子）👆クレジットに子の名前（クレジット）]
        A3 --> A4[（子）❤️Run → エンディング到達（ゲーム中）]
    end
    subgraph L3["L3: 社会的価値（友達への披露）"]
        A5[（子）👆友達に「クリアしたから見て！」（LINE）]
        A5 --> A6[（友達）❤️「すごい、最後まで作ったの？」（スマホブラウザ）]
    end
    subgraph L4["L4: 人生変容価値（家族 identity の頂点）"]
        A7[（家族）❤️「このゲーム、ぼくたちが作ったんだ」という共有体験が家族の物語として残る（対面）]
    end

    A2 --> A3
    A4 --> A5
    A6 --> A7
```

> **この瞬間に立ち上がった価値**：CJ30 は CQP の Layer 4 が最も濃く立ち上がる瞬間。**プロダクトの存在意義を集約する**ジャーニー。

### CJ35-v5: AI で修正したらエラーが出て動かない

**価値階層的な意味**：ガードレールは L1 の機能だが、家族関係（L4）を守るための仕組み。

```mermaid
flowchart LR
    subgraph L1["L1: 機能的価値（ヘッドレス検証）"]
        A1[（親）👆AIに依頼（AI入力画面）]
        A1 --> A2[（ビルド）👀ヘッドレス検証 NG → 開発版を出さない（承認キュー）]
    end
    subgraph L2["L2: 情緒的価値（安心感）"]
        A3[（子）❤️常に動くゲームだけが届く（ゲーム中）]
    end
    subgraph L3["L3: 社会的価値"]
        A4[（子）👆友達にも自信を持って URL を送れる（LINE）]
    end
    subgraph L4["L4: 人生変容価値（信頼の蓄積）"]
        A5[（家族）❤️失敗時の振る舞いで親への信頼が揺らがない／家族関係が摩耗しない（対面）]
    end

    A2 --> A3
    A3 --> A4
    A4 --> A5
```

> **この瞬間に立ち上がった価値**：ガードレールの本当の役割は L4。L1 のヘッドレス検証は手段であって目的ではない。**目的は家族関係の保護**。

### CJ43-v5: 実公開で遊ばれた記録が見える

**価値階層的な意味**：単なるアクセスログ（L1）が、家族の物語の素材（L4）になる。

```mermaid
flowchart LR
    subgraph L1["L1: 機能的価値（ログ）"]
        A1[（友達）👆実公開URLを開く（スマホブラウザ）]
        A1 --> A2[（親）👀日別・ページ別ログ（観察者DB）]
    end
    subgraph L2["L2: 情緒的価値（事実で安心）"]
        A3[（親）❤️「遊ばれてた」と分かる安心感（対面）]
    end
    subgraph L3["L3: 社会的価値（届いた／届かなかった）"]
        A4[（親子）👆届かなかった場合は公開経路を見直す（対面）]
    end
    subgraph L4["L4: 人生変容価値（家族の振り返り）"]
        A5[（家族）❤️「あの月は ◯ ちゃんがよく遊んでくれた」が家族の記憶として残る（履歴UI）]
    end

    A2 --> A3
    A3 --> A4
    A4 --> A5
```

> **この瞬間に立ち上がった価値**：ログという機能が、半年後の振り返りで「家族の物語」の一部になる。

### CJ45-v5: 関係の摩耗（v3 で新規追加、L4 を直接守るジャーニー）

**価値階層的な意味**：このジャーニーは L4 を直接守る。L1-L3 をスキップして L4 から書く稀なケース。

```mermaid
flowchart LR
    subgraph L1["L1: 機能的価値（却下が成立する）"]
        A1[（システム）👆「却下」が常に成立する保証（承認キュー）]
    end
    subgraph L2["L2: 情緒的価値（引き下がる余地）"]
        A2[（親）👆「君が決めたほうがいい」と引き下がれる（対面）]
    end
    subgraph L3["L3: 社会的価値"]
        A3[（家族）👆友達への配慮もしつつ家族の判断を優先（対面）]
    end
    subgraph L4["L4: 人生変容価値（家族関係の保全）"]
        A4[（家族）❤️衝突しても関係が摩耗しない／翌日また開ける（履歴UI）]
    end

    A1 --> A2
    A2 --> A3
    A3 --> A4
```

> **この瞬間に立ち上がった価値**：「却下が成立する」という機能が L4（家族関係の保全）の本体。L4 のために L1 を設計するパターン。

## このバージョンを採用するときに変わること

- 全ジャーニーが **L1 → L2 → L3 → L4** の流れで描かれる
- 価値階層別 KPI ダッシュボード（親(観察者) ダッシュボードに統合）
- マーケティング素材が層別に分かれ、**最終訴求は L4**（家族の物語）
- 競合分析が層別に整理され、L4 まで届くプロダクトが CQP だけであることを示せる
- 価値階層モデルが**プロダクトの組織哲学**に降りる：「私たちは家族の物語を作るプロダクトです」
