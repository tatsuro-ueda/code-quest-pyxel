# 実験案 v2：カスタマージャーニー ── 多人格分解軸

> 実験ラベル：**v2 / 多人格分解軸**
> 作成日：2026-04-25
> 視点：1 つのジャーニーの中で**人格が切り替わる**さまを明示する。同じ「子ども」が `子(プレイヤー)` →`子(批評家)` →`子(社会人)` と移ろう。同じ「親」が `親(翻訳者)` →`親(セーフティネット)` →`親(観察者)` と移ろう。**人格の切り替わり点こそ価値の発生点**である。
> 根拠：[`experimental-customer-jobs-v2.md`](./experimental-customer-jobs-v2.md)

---

## 凡例（v2 固有）

- ノード形式：`[（人格）絵文字 文（タッチポイント）]`
- 人格は `子(プレイヤー)` `子(クリエイター)` `子(批評家)` `子(社会人)` `親(翻訳者)` `親(セーフティネット)` `親(観察者)` `友達` のいずれか
- **subgraph の分け方を変える**：Before / After ではなく、**人格の遷移**で区切る
- 各 mermaid の右側に、**人格遷移サマリー**を 1 行で添える

### CJ08-v2: 敵が強すぎる

**人格遷移**：`子(プレイヤー) → 親(観察者) → 親(翻訳者) → 親(セーフティネット) → 子(批評家) → 子(プレイヤー)`

子の中の「プレイヤー」と「批評家」、親の中の「観察者」「翻訳者」「セーフティネット」が連携して 1 周のループを閉じる。

**感情**：❌怒り（プレイヤー） →❤️判断の手応え（批評家） → 続行できる楽しさ（プレイヤー）

```mermaid
flowchart TD
    P1[子(プレイヤー)<br/>💢「敵強すぎ！」（ゲーム中）]
    P1 --> P2[親(観察者)<br/>👀今日の集中度を見て、いま受け止めるべきと判断（対面）]
    P2 --> P3[親(翻訳者)<br/>👆AIに「スライムのHP 50→30 にして」（AI入力画面）]
    P3 --> P4[親(セーフティネット)<br/>👀ヘッドレス検証 → 開発版カードにのみ反映（承認キュー）]
    P4 --> P5[子(批評家)<br/>👀開発版で 2 回／本番で 4 回、体感差を比較（ゲーム中）]
    P5 --> P6[子(批評家)<br/>❤️「開発版のほう！」承認（選択ページ）]
    P6 --> P7[子(プレイヤー)<br/>❤️続きを楽しめる（ゲーム中）]

    classDef child fill:#fff3cd,stroke:#856404,color:#000000;
    classDef parent fill:#d1ecf1,stroke:#0c5460,color:#000000;
    classDef end fill:#d4edda,stroke:#155724,color:#000000;

    class P1,P5,P6,P7 child;
    class P2,P3,P4 parent;
```

> **このジャーニーの構造的意味**：プレイヤー人格の苦痛が、「翻訳→検証→批評→決定→再開」の人格リレーを通って解消される。途中で**親(観察者)**が入っている点が重要——親が観察者人格でいないと、翻訳者人格が暴走しやすい。

### CJ31-v2: 子どもが変更を承認する（多人格版）

**人格遷移**：`親(翻訳者) → 親(セーフティネット) → 子(批評家)（×複数回） → 子(プレイヤー)`

**感情**：❌親が決めたものを渡される →❤️批評家として体感差で判断する

```mermaid
flowchart TD
    A1[親(翻訳者)<br/>👆AIに依頼（AI入力画面）]
    A1 --> A2[親(セーフティネット)<br/>👀ビルド検証通過 → 開発版に反映（承認キュー）]
    A2 --> A3[子(批評家)<br/>👀開発版で遊ぶ HP=30 → 2 かいで たおせる（ゲーム中）]
    A3 --> A4[子(批評家)<br/>👀本番で遊ぶ HP=50 → 4 かい かかる（ゲーム中）]
    A4 --> A5[子(批評家)<br/>❤️体感差で「開発版のほう！」承認（選択ページ）]
    A5 --> A6[子(プレイヤー)<br/>❤️新しいバランスで遊び続ける（ゲーム中）]

    classDef child fill:#fff3cd,stroke:#856404,color:#000000;
    classDef parent fill:#d1ecf1,stroke:#0c5460,color:#000000;

    class A3,A4,A5,A6 child;
    class A1,A2 parent;
```

> **このジャーニーの構造的意味**：「子(批評家)」がフローのほぼ半分を占めている。**承認キューは批評家人格のためのプロダクト**。

### CJ22-v2: 友達のフィードバックを反映する

**人格遷移**：`友達 → 子(社会人) → 親(観察者) → 子(社会人) → 親(翻訳者) → 親(セーフティネット) → 子(批評家) → 子(社会人) → 友達`

**感情**：❌反応を受け取る側に留まる →❤️家族の好循環に持ち帰る・閉じる

```mermaid
flowchart TD
    F1[友達<br/>💡「ここ難しい」（スマホブラウザ）]
    F1 --> S1[子(社会人)<br/>👂友達の感想を受け止める（対面）]
    S1 --> O1[親(観察者)<br/>👀子の今の状態と関心を察する（対面）]
    O1 --> S2[子(社会人)<br/>👆「パパ、これ直して！」と判断（対面）]
    S2 --> T1[親(翻訳者)<br/>👆AIに「ここを楽にして」と翻訳（AI入力画面）]
    T1 --> N1[親(セーフティネット)<br/>👀ビルド検証通過 → 開発版に反映（承認キュー）]
    N1 --> C1[子(批評家)<br/>👀遊び比べて承認（選択ページ）]
    C1 --> S3[子(社会人)<br/>👆友達に URL 再送（LINE）]
    S3 --> F2[友達<br/>❤️「お、いい感じ！」フィードバックループ完結（スマホブラウザ）]

    classDef child fill:#fff3cd,stroke:#856404,color:#000000;
    classDef parent fill:#d1ecf1,stroke:#0c5460,color:#000000;
    classDef friend fill:#f8d7da,stroke:#721c24,color:#000000;

    class S1,S2,S3,C1 child;
    class O1,T1,N1 parent;
    class F1,F2 friend;
```

> **このジャーニーの構造的意味**：このジャーニーは家族の中で**最も多くの人格を経由する**。最初と最後が「友達」で、家庭の外まで好循環が貫通している。途中の `S2` が一番繊細——親(観察者)がフィードバックを子(社会人)に渡すかどうかの**判断点**。ここを誤ると「親が代理判断」のアンチパターンに陥る。

### CJ26-v2:「自分たちのゲーム」と言えるようになる

**人格遷移**：`子(クリエイター) ↔ 親(翻訳者)（往復）→ 親(観察者) → 子(社会人) → 子(プレイヤー)`

**感情**：❌外部プロダクトの色が抜けない →❤️「ぼくたちのゲーム」と言えるアイデンティティ

```mermaid
flowchart TD
    A1[子(クリエイター)<br/>👆Tilemap でマップを書き換え（リソースエディター）]
    A1 --> A2[親(翻訳者)<br/>👆AIに「敵やセリフを足して」（AI入力画面）]
    A2 --> A3[子(クリエイター)<br/>👆Sprite/Sound を Code Maker で触る（Pyxel Code Maker）]
    A3 --> A4[親(翻訳者)<br/>👆AIに「BGM を取り込んで」（AI入力画面）]
    A4 --> A5[親(観察者)<br/>👀オリジナル部分が半分を超えたことに気づく（対面）]
    A5 --> A6[子(社会人)<br/>👆「友達に見せたい！」と発露（対面）]
    A6 --> A7[子(プレイヤー)<br/>❤️Run → エンディング到達「ぼくたちのゲーム」（ゲーム中）]

    classDef child fill:#fff3cd,stroke:#856404,color:#000000;
    classDef parent fill:#d1ecf1,stroke:#0c5460,color:#000000;

    class A1,A3,A6,A7 child;
    class A2,A4,A5 parent;
```

> **このジャーニーの構造的意味**：「自分たちのゲーム」感は、**子(クリエイター)** が表層（見た目・音）まで触れた瞬間に閾値を越える。親(観察者) はそれを言語化せず、ただ気づく役。

### CJ35-v2: AI で修正したらエラーが出て動かない（多人格版）

**人格遷移**：`子(プレイヤー) → 親(翻訳者) → 親(セーフティネット)（強烈に作動） → 子(プレイヤー)`

**感情**：❌壊れた版が届く（プレイヤーが集中を切る） →❤️届かない（プレイヤーが守られる）

```mermaid
flowchart LR
    subgraph BeforeFlow["Before"]
        BP1[子(プレイヤー)<br/>💢「新しい技ほしい！」（対面）]
        BP1 --> BT1[親(翻訳者)<br/>👆AIに依頼（AI入力画面）]
        BT1 --> BB[子(プレイヤー)<br/>❌Run → 黒い画面 → 集中が切れる（ゲーム中）]
    end
    subgraph AfterFlow["After"]
        AP1[子(プレイヤー)<br/>💡「新しい技ほしい！」（対面）]
        AP1 --> AT1[親(翻訳者)<br/>👆AIに依頼（AI入力画面）]
        AT1 --> AN1[親(セーフティネット)<br/>👀ヘッドレス検証 NG → 開発版を出さず親にエラー要約（ビルドログ）]
        AN1 --> AT2[親(翻訳者)<br/>👆AIに再依頼（AI入力画面）]
        AT2 --> AN2[親(セーフティネット)<br/>👆OK → 開発版カードに反映（選択ページ）]
        AN2 --> AP2[子(プレイヤー)<br/>❤️常に動くゲームだけが届く（ゲーム中）]
    end

    classDef child fill:#fff3cd,stroke:#856404,color:#000000;
    classDef parent fill:#d1ecf1,stroke:#0c5460,color:#000000;

    class BP1,BB,AP1,AP2 child;
    class BT1,AT1,AN1,AT2,AN2 parent;
```

> **このジャーニーの構造的意味**：ここでは **親(セーフティネット)** が最も強く作動する。「親(セーフティネット) を機能させる装置」がガードレール CJ35-CJ41 の正体。

### CJ34-v2: 承認したあとに「やっぱり」となる

**人格遷移**：`子(批評家) → 子(プレイヤー) → 子(批評家)（再判断） → 親(翻訳者) → 子(批評家)`

**感情**：❌また親任せ →❤️批評家として戻すことを自分で決められる

```mermaid
flowchart TD
    A1[子(批評家)<br/>👆承認して反映（選択ページ）]
    A1 --> A2[子(プレイヤー)<br/>👆しばらく遊んで違和感に気づく（ゲーム中）]
    A2 --> A3[子(批評家)<br/>💡「やっぱり前のがよかった！」（対面）]
    A3 --> A4[親(翻訳者)<br/>👆「もどして」依頼を承認キューに登録 HP: 30→50（承認キュー）]
    A4 --> A5[子(批評家)<br/>❤️自分で承認して元に戻る（選択ページ）]

    classDef child fill:#fff3cd,stroke:#856404,color:#000000;
    classDef parent fill:#d1ecf1,stroke:#0c5460,color:#000000;

    class A1,A2,A3,A5 child;
    class A4 parent;
```

> **このジャーニーの構造的意味**：「批評家」は二度判断する権利を持つ。一度目の判断で固定されないことが、批評家人格を**プロダクトの主役**として扱うということ。

### CJ20-v2: 演出の有無で印象が変わることを体験する

**人格遷移**：`親(観察者) → 親(翻訳者) → 子(プレイヤー) → 子(クリエイター)`

**感情**：❌一方的な説明 →❤️プレイヤー人格として体感し、クリエイター人格として議論したくなる

```mermaid
flowchart TD
    A1[親(観察者)<br/>👀演出の価値を口で説明しても伝わらない、と気づく（対面）]
    A1 --> A2[親(翻訳者)<br/>👆「音と光をオフにして遊んでみて」（対面）]
    A2 --> A3[子(プレイヤー)<br/>👆無音・演出なしでプレイ →「なんか寂しい」（ゲーム中）]
    A3 --> A4[親(翻訳者)<br/>👆ON に戻す（対面）]
    A4 --> A5[子(プレイヤー)<br/>❤️「全然違う！こっちがいい！」（ゲーム中）]
    A5 --> A6[子(クリエイター)<br/>💡「次にどんな演出を足したい？」議論（対面）]

    classDef child fill:#fff3cd,stroke:#856404,color:#000000;
    classDef parent fill:#d1ecf1,stroke:#0c5460,color:#000000;

    class A3,A5,A6 child;
    class A1,A2,A4 parent;
```

> **このジャーニーの構造的意味**：プレイヤー人格の体感を経由してから、クリエイター人格の議論が始まる。**人格の通過順序**が学習効果を決める。

### CJ43-v2: 実公開で遊ばれた記録が見える

**人格遷移**：`子(社会人) → 友達 → 親(観察者) → 親(観察者)（事実で見直す）`

**感情**：❌勘で判断 →❤️親(観察者) が事実で意思決定できる

```mermaid
flowchart TD
    A1[子(社会人)<br/>👆URL を送る（LINE）]
    A1 --> A2[友達<br/>👆実公開 URL を開いて遊ぶ（スマホブラウザ）]
    A2 --> A3[親(観察者)<br/>👀日別・ページ別でアクセスログを確認（実公開アクセスログ）]
    A3 --> A4[親(観察者)<br/>❤️遊ばれていれば嬉しい／なければ公開経路を見直せる（対面）]

    classDef child fill:#fff3cd,stroke:#856404,color:#000000;
    classDef parent fill:#d1ecf1,stroke:#0c5460,color:#000000;
    classDef friend fill:#f8d7da,stroke:#721c24,color:#000000;

    class A1 child;
    class A3,A4 parent;
    class A2 friend;
```

> **このジャーニーの構造的意味**：このジャーニーは**親(観察者)** のためのプロダクト。実公開アクセスログは「親(観察者) ダッシュボード」の最初の機能。今後、修正サイクル時間や子の集中時間も同じダッシュボードに乗るとよい。

## このバージョンを採用するときに変わること

- 全 42 ジャーニーが**人格遷移図**として書かれる（subgraph Before/After ではなく、人格でクラスタリング）
- 一覧表に「主要人格」「人格遷移パターン」列が増える
- **親(観察者) ダッシュボード**が新規機能領域として浮上
- 「役割固定化を防ぐ UI」がプロダクトの核機能になる
- マーケティング素材は「親も子も、その瞬間ごとに何の人格でいるかを問い直すプロダクト」として打ち出す

## 残り 34 本の方針

各ジャーニーで「主要人格」と「人格遷移パターン」を特定し、人格ごとの色分け（子＝黄、親＝青、友達＝赤）でクラスタリングして書く。**人格の切り替わり点に絵文字の感情変化が起きる**ように設計するのが鍵。
