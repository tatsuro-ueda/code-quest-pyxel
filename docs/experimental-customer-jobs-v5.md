# 実験案 v5：顧客のジョブ ── 価値階層軸（4 層モデル）

> 実験ラベル：**v5 / 価値階層軸**
> 作成日：2026-04-25
> 視点：プロダクトが提供する価値を **4 階層** で分解し、各階層に対応するジョブを定義する。CQP は表層の「機能的価値」だけでなく、最深層の「人生変容的価値（family identity の形成）」まで届くプロダクトとして再定義する。
> 動機：v1（外）／v2（内）／v3（事故）／v4（時間）に対して、v5 は「**価値の深さ**」軸。「速い」「便利」を売るプロダクトとは違う層で勝負する。

---

## 0. 価値階層モデル（Bain Elements of Value を CQP 向けに再構成）

> 一般的な価値階層は「機能的 < 情緒的 < 人生変容的 < 社会的影響」の 4 層。CQP では「家族」という単位を扱うため、**家族 identity** の形成を頂点に置く独自の階層を定義する。

```
┌─────────────────────────────────────────────────┐
│ Layer 4: 人生変容価値                                │
│   家族の identity / 子の自己効力感 / 親子の対等関係         │
├─────────────────────────────────────────────────┤
│ Layer 3: 社会的価値                                 │
│   友達コミュニティ / 「みんなで作った」感 / 誇り             │
├─────────────────────────────────────────────────┤
│ Layer 2: 情緒的価値                                 │
│   集中の連続 / 所有感 / 安心感（壊れない） / 達成感           │
├─────────────────────────────────────────────────┤
│ Layer 1: 機能的価値                                 │
│   速さ / 動く / 配信 / バグ修正 / データ変更                 │
└─────────────────────────────────────────────────┘
```

**重要**：上の層は下の層の上に成立する。だが**プロダクトの差別化はほぼ常に上の層で起きる**。Layer 1 だけのプロダクトは無数にある。Layer 4 まで届くプロダクトは稀。

---

## 1. 各層のジョブ

### Layer 1：機能的価値のジョブ

> **VL1.1**：子の要望を AI が動くコードに翻訳する  
> **VL1.2**：壊れた版を子に届けない  
> **VL1.3**：友達のスマホに URL で配信する  
> **VL1.4**：データを SSoT から自動生成する  
> **VL1.5**：見た目・音を直接エディタで触る

**競合との比較**：このレイヤーは vibe coding、Cursor、Roblox Studio、RPG Maker MV など多数のプロダクトが部分的に提供する。**Layer 1 だけでは差別化困難**。

---

### Layer 2：情緒的価値のジョブ

> **VL2.1**：子の集中が切れない（数分で 1 周）  
> **VL2.2**：「自分のキャラ」「自分のマップ」と言える所有感  
> **VL2.3**：「壊れない」という安心感  
> **VL2.4**：「直った！」という達成感の連続  
> **VL2.5**：自分の判断で決めた手応え（体感差での承認）

**競合との比較**：Scratch は VL2.2 を強く提供するが VL2.1（速さ）が子単独依存／Roblox は VL2.4 を提供するが VL2.3（壊れない）が弱い。**CQP は VL2.1-VL2.5 を全て同時に提供することで差別化**。

---

### Layer 3：社会的価値のジョブ

> **VL3.1**：友達に見せて「すごい」と言われる誇り  
> **VL3.2**：友達のフィードバックを取り込んで一緒に作っている感覚  
> **VL3.3**：「自分たちのゲーム」と外向きに言える形  
> **VL3.4**：友達のフィードバックを子が judge できる主体性

**競合との比較**：Roblox は VL3.1 を巨大コミュニティで提供するが、その代償に課金経済と社会圧。Scratch はリミックス文化で VL3.2 を提供するが配信が PC 寄り。**CQP は家庭中心・URL 配信で「同心円的に広がる小さな社会」を作る**。

---

### Layer 4：人生変容価値のジョブ

> **VL4.1**：「家族で作った作品がある」という家族 identity の形成  
> **VL4.2**：「自分は決められる人だ」という子の自己効力感  
> **VL4.3**：親子の対等関係（先生／生徒ではない）の体験的学習  
> **VL4.4**：失敗・後悔・修復のプロセスを共有体験として持つ  
> **VL4.5**：完成した作品を経て「次は何を作ろうか」と未来を見る家族の習慣

**競合との比較**：このレイヤーを意識的に設計しているクリエイティブ・プロダクトは**ほぼ存在しない**。Roblox は子個人のスキル獲得を売り、Scratch は教育を売る。**CQP は「家族の物語」を売る稀有なプロダクト**。

## 2. 各層の核心問い

各層が満たされているかを問う 1 行の指標。

| 層 | 核心問い |
|---|---|
| Layer 1 | このプロダクトで「**やりたいこと**」が技術的にできるか？ |
| Layer 2 | このプロダクトを使っている**今この瞬間**、家族は気持ちよく過ごしているか？ |
| Layer 3 | このプロダクトを通じて、家族は**外の人と良い関わり**ができているか？ |
| Layer 4 | このプロダクトを使い続けた結果、**家族として何が育ったか**？ |

## 3. 既存ジョブを 4 層に再マッピング

| 既存ジョブ | 主に提供する層 | 補助的に提供する層 |
|---|---|---|
| Job1（コミュニケーション） | Layer 4（対等関係） | Layer 2 |
| Job2（好循環） | Layer 1, 2 | Layer 3, 4 |
| Job3（作品レベル） | Layer 3 | Layer 2, 4 |
| Job4（ゲームを楽しむ） | Layer 2 | Layer 4 |
| Job5（プログラミング学習） | Layer 1 | Layer 2 |

**気付き**：既存ジョブの大半は Layer 1, 2 にフォーカスしている。Layer 3 は Job3 が部分的にカバー。**Layer 4 は Job1 でしか言及されていない**。プロダクトの真の差別化が起きる Layer 4 が、ジョブとして弱い。

---

## 4. Layer 4（人生変容）を一級ジョブに昇格する

### 新ジョブ：Job-LV1「家族 identity の形成」

> 家族として、「あの夏に一緒にゲームを作った」「あのキャラの名前は子どもがつけた」「あのエンディングは親子で書いた」という**共有された記憶**を持つ。これが家族としての自己像の一部になる。

**設計に降ろすと**：
- 履歴 UI（ぶれた跡を進化として残す）
- クレジット表示（CJ30）
- 作品の archive（半年後に振り返れる）
- 友達からの反応の記録（「あの時 ◯◯ ちゃんが楽しんでくれた」）

---

### 新ジョブ：Job-LV2「子の自己効力感」

> 子が「自分は意思決定できる人だ」「自分の判断は尊重される」と感じる体験を、繰り返しの中で蓄積する。

**設計に降ろすと**：
- 承認キュー（CJ31, CJ32, CJ33, CJ34）の積み重ね
- 「もどして」も承認できる権利
- 友達フィードバックを子が judge できる構造（CJ22）

---

### 新ジョブ：Job-LV3「親子の対等関係の体験的学習」

> 「先生／生徒」ではなく「仲間」として作る経験を**繰り返す**ことで、家族の関係様式そのものが変わる。

**設計に降ろすと**：
- 役割分担表（親 = 翻訳、子 = ハンドル）
- 役割交代ジャーニー（CJ25）
- 親(セーフティネット) のセルフ・ガード機構

---

### 新ジョブ：Job-LV4「失敗の共有体験」

> 「AI が壊した時、親が暴走した時、友達が遊んでくれなかった時」を、家族が**一緒に経験して乗り越える**。失敗体験こそ家族の絆を作る。

**設計に降ろすと**：
- 失敗時のジャーニー（v3 の F1-F6）
- 履歴 UI で「やり直し」「戻し」「諦め」が**進化として**表示

---

### 新ジョブ：Job-LV5「次の物語への接続」

> ある作品を完成させたあと、「次は何を作ろうか」と家族が自然に思える。プロダクトが「終わり」ではなく「次への入り口」として機能する。

**設計に降ろすと**：
- エンディング到達後のジャーニー（CJ30 の延長）
- 完成作品の archive とテンプレ化
- 弟妹・別プロジェクトへの継承

## 5. 価値階層別の優先設計

| 層 | 優先設計 | KPI |
|---|---|---|
| L1 | AI 翻訳 + ヘッドレス検証 + 配信 | サイクル時間、起動成功率、エラー率 |
| L2 | 承認キュー、リソースエディター、ガードレール | 1 セッション周回数、所有感アンケート |
| L3 | URL 即プレイ、フィードバックループ、アクセスログ | 友達プレイ率、フィードバック反映率 |
| L4 | 履歴 UI、クレジット、archive、観察者ダッシュボード | 継続月数、家族関係スコア、再開率 |

---

## 6. 価値階層を採用すると変わること

- **マーケティングが層別に分かれる**
  - Layer 1 訴求：「AI で 5 分で修正」
  - Layer 2 訴求：「子の集中が切れない」
  - Layer 3 訴求：「友達と一緒に作るループ」
  - Layer 4 訴求：「**家族の物語が育つ**」（差別化の本丸）
- **価値の深さで料金体系を組める**（Layer 4 は有料、L1-L3 は無料）
- **競合分析が層別に整理される**：競合は L1, L2 までしか届いていない
- **新規ジャーニー追加**：CJ44（関心の蒸発、L4 と連動）、CJ45（関係の摩耗、L4 と連動）、CJ46（はじめて起動、L1 の入り口）

## 7. 価値階層の戦略的含意

### 含意 1：Layer 1 は「コモディティ」化が早い

vibe coding が成熟すれば、AI 修正の速さは誰でも提供できる。**Layer 1 だけで戦うと 2 年で陳腐化**する。

### 含意 2：Layer 4 は「真似しにくい」

家族 identity の形成は、プロダクトの哲学・UI・コミュニケーション設計が一貫していて初めて生まれる。**Layer 4 で差別化すれば長期的な堀になる**。

### 含意 3：Layer 4 を意識すると Layer 1 の作り方が変わる

例：Layer 4 を満たすには「履歴」が必要 → Layer 1 のコミット粒度が変わる（変更単位を小さく保つ） → Layer 1 の AI 翻訳の作法が変わる（1 回の AI 依頼で 1 つの変更）

### 含意 4：失敗時こそ Layer 4 が試される

家族 identity は順調な時には育たない。**失敗時に「一緒に乗り越えた」体験**が identity を作る。これが v3 の失敗ジョブと整合する。
