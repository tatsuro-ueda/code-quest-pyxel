# 実験案 v6：カスタマージャーニー ── アンチジョブ軸（境界を守るシーン）

> 実験ラベル：**v6 / アンチジョブ軸**
> 作成日：2026-04-25
> 視点：ジャーニーを「**境界が試される瞬間**」として描く。ある瞬間、プロダクトは便利さの誘惑にかられる。その時、**境界を守るかどうか**が価値の発生点になる。
> 根拠：[`experimental-customer-jobs-v6.md`](./experimental-customer-jobs-v6.md)

---

## 凡例（v6 固有）

- **subgraph の構造**：
  - `Temptation`：便利さ・機能追加の誘惑が生じる瞬間
  - `Crossing the line`：境界を越えてしまった場合に起きる悪化（反面教師）
  - `Holding the line`：境界を守った場合の体験
- mermaid のラベルに**アンチジョブ番号 (AJ1-AJ7)** を明示

---

## 代表ジャーニー：境界が試される 7 つの瞬間

---

### CJ-AJ1: 「学習進捗を見たい」と親が思った瞬間

**境界**：AJ1（学習効率を最大化しない、評価しない）

```mermaid
flowchart LR
    subgraph Temptation["誘惑：親が学習進捗を見たくなる"]
        T1[（親）❓「うちの子、どれくらい学んだんだろう」（対面）]
        T1 --> T2[（親）💦進捗ダッシュボードを欲しがる（観察者DB）]
    end
    subgraph Crossed["越えた場合"]
        C1[（プロダクト）👆スキルレベル UI を実装する（観察者DB）]
        C1 --> C2[（親）👀子の進捗を気にし始める（対面）]
        C2 --> C3[（子）❌「学ばないとダメ」と感じる（対面）]
        C3 --> C4[（家族）❌先生／生徒化（対面）]
    end
    subgraph Holding["守った場合"]
        H1[（プロダクト）👆進捗 UI を作らない／作るなら最終プレイ日時のみ（観察者DB）]
        H1 --> H2[（親）💡「学んだか」より「楽しんだか」を見る視点に（対面）]
        H2 --> H3[（家族）❤️プロダクトを使う関係が「学校」化しない（対面）]
    end
```

> **境界を守ることで生まれる価値**：プロダクトを「学習ツール」ではなく「**家族の遊び場**」として保てる。

---

### CJ-AJ2: 「友達の反応をランキング化したい」と思った瞬間

**境界**：AJ2（評価をプロダクトに持ち込まない）

```mermaid
flowchart LR
    subgraph Temptation["誘惑：友達の反応を可視化したい"]
        T1[（子）💡「友達が遊んでくれた数を知りたい」（対面）]
        T1 --> T2[（プロダクト企画）💦「友達順ランキング」機能を提案（企画書）]
    end
    subgraph Crossed["越えた場合"]
        C1[（プロダクト）👆ランキングを実装（選択ページ）]
        C1 --> C2[（子）❌「◯◯ ちゃんは ◯ 回も遊んでくれたのに、◯◯ ちゃんは…」（対面）]
        C2 --> C3[（子）❌他人の反応を気にして「自分の好み」が薄まる（対面）]
    end
    subgraph Holding["守った場合"]
        H1[（プロダクト）👆ランキングを実装しない／実公開アクセスログだけ親(観察者) が見る（観察者DB）]
        H1 --> H2[（子）👆友達の対面・LINE 反応で十分受け取れる（LINE）]
        H2 --> H3[（家族）❤️作品が「比較対象」ではなく「自分たちのもの」のままでいる（対面）]
    end
```

> **境界を守ることで生まれる価値**：作品が**比較財**にならない。所有感が侵食されない。

---

### CJ-AJ3: 「子のゲームを売れる仕組みを」と提案された瞬間

**境界**：AJ3（課金経済を作らない）

```mermaid
flowchart LR
    subgraph Temptation["誘惑：マネタイズ機能の提案"]
        T1[（プロダクト企画）💡「子の作品を売れるようにしたら大ヒット」（企画書）]
        T1 --> T2[（投資家）💡「Roblox のように子が稼げるエコシステム」（提案書）]
    end
    subgraph Crossed["越えた場合"]
        C1[（プロダクト）👆販売機能・仮想通貨実装（販売ページ）]
        C1 --> C2[（子）❌「売れるゲーム」を作ろうとする（対面）]
        C2 --> C3[（子）❌完成しないと意味がない圧 → 80% 未完成のアンチパターンへ（対面）]
        C3 --> C4[（家族）❌商業的失敗が家族関係に持ち込まれる（対面）]
    end
    subgraph Holding["守った場合"]
        H1[（プロダクト）👆販売機能を作らない／作品は家族と友達で楽しむのが完結点（対面）]
        H1 --> H2[（子）❤️完成しなくても許される／途中の試作も価値があるとされる（対面）]
        H2 --> H3[（家族）❤️商業的プレッシャーから自由な創作（対面）]
    end
```

> **境界を守ることで生まれる価値**：「未完成」が許される空間。失敗しても家族関係に響かない。

---

### CJ-AJ4: 「世界中のユーザーに作品を届けたい」と提案された瞬間

**境界**：AJ4（友達コミュニティを大規模化しない）

```mermaid
flowchart LR
    subgraph Temptation["誘惑：世界中に拡散できる仕組み"]
        T1[（プロダクト企画）💡「公開検索＋世界ランキング」（企画書）]
    end
    subgraph Crossed["越えた場合"]
        C1[（プロダクト）👆世界中のユーザーが見られる公開ページ実装（公開ページ）]
        C1 --> C2[（子）❌「世界中の作品」と比較される（対面）]
        C2 --> C3[（子）❌「自分のクオリティ低い」と感じて創作意欲が萎む（対面）]
    end
    subgraph Holding["守った場合"]
        H1[（プロダクト）👆URL 共有は親子が選んだ友達のみ／公開検索なし（LINE）]
        H1 --> H2[（家族）❤️「内輪の楽しさ」が守られる（対面）]
    end
```

> **境界を守ることで生まれる価値**：内輪の温度感。外の評価軸に晒されない安全圏。

### CJ-AJ5: 「親に詳細な管理ダッシュボードを」と提案された瞬間

**境界**：AJ5（親に過剰な管理機能を渡さない）

```mermaid
flowchart LR
    subgraph Temptation["誘惑：親への透明性を最大化"]
        T1[（親）❓「子が何をしているか全部見たい」（対面）]
        T1 --> T2[（プロダクト企画）💡「全編集ログ・全 AI 依頼を親に開示」（企画書）]
    end
    subgraph Crossed["越えた場合"]
        C1[（プロダクト）👆全ログを親に開示（観察者DB）]
        C1 --> C2[（親）👀子の全行動を監視（対面）]
        C2 --> C3[（子）❌プライバシーがない／親の目を気にする（対面）]
        C3 --> C4[（家族）❌「監視される側／監視する側」関係（対面）]
    end
    subgraph Holding["守った場合"]
        H1[（プロダクト）👆観察者ダッシュボードは「壊れた版が届かないか」「関心が蒸発してないか」だけ（観察者DB）]
        H1 --> H2[（親）❤️必要最小限の情報で介入判断（対面）]
        H2 --> H3[（子）❤️プライバシーが守られる（対面）]
    end
```

> **境界を守ることで生まれる価値**：「観察」と「監視」を区別する。親が**家族関係を守る**のに必要な情報だけ。

### CJ-AJ6: 「AI に最適なバランスを判断させたい」と提案された瞬間

**境界**：AJ6（AI に「正解」を判断させない）

```mermaid
flowchart LR
    subgraph Temptation["誘惑：AI が判断者になる便利さ"]
        T1[（プロダクト企画）💡「AI が最適な敵 HP を計算」（企画書）]
        T1 --> T2[（親）💡「子に聞かなくても AI が最適化してくれる」（対面）]
    end
    subgraph Crossed["越えた場合"]
        C1[（プロダクト）👆AI が「推奨設定」を画面に出す（AI出力画面）]
        C1 --> C2[（子）❌「AI のおすすめ」を選ぶ→自分の体感より AI の判断（選択ページ）]
        C2 --> C3[（子）❌自己効力感（自分は決められる）が育たない（対面）]
    end
    subgraph Holding["守った場合"]
        H1[（プロダクト）👆AI は複数案を並列で出すだけ／推奨は出さない（AI出力画面）]
        H1 --> H2[（子）👆「開発版／本番」を遊び比べて体感で判断（選択ページ）]
        H2 --> H3[（子）❤️判断する経験が積み上がる（履歴UI）]
    end
```

> **境界を守ることで生まれる価値**：AI を「翻訳機」に留め、判断者にしない。子の自己効力感が育つ余地を残す。

### CJ-AJ7: 「親が直接見た目を編集できる UI が便利」と提案された瞬間

**境界**：AJ7（親がリソースを直接編集しない）

```mermaid
flowchart LR
    subgraph Temptation["誘惑：親が直接触ったほうが速い"]
        T1[（親）💡「子が忙しいから、私が代わりにスプライトを描こう」（対面）]
        T1 --> T2[（プロダクト企画）💡「親モード でリソース直接編集」（企画書）]
    end
    subgraph Crossed["越えた場合"]
        C1[（親）👆親モードでスプライトを描く（リソースエディター）]
        C1 --> C2[（子）❌「自分のキャラ」感が薄まる（ゲーム中）]
        C2 --> C3[（家族）❌Job-LV2（自己効力感）が崩れる（対面）]
    end
    subgraph Holding["守った場合"]
        H1[（プロダクト）👆親モードを作らない／親はリソースを直接触らない（プロダクト哲学）]
        H1 --> H2[（親子）👆Code Maker を一緒に開いて子が描く・親はそばにいる（Pyxel Code Maker）]
        H2 --> H3[（子）❤️「自分が描いた」が保たれる（ゲーム中）]
    end
```

> **境界を守ることで生まれる価値**：**親の便利さ**より**子の所有感**を優先する明確な意思決定。

## 全アンチジョブの集約：CJ-AJ-MASTER

「境界を守る」という哲学そのものをジャーニー化する。

```mermaid
flowchart TD
    Start[（プロダクトチーム）💡新機能の提案] --> Filter{境界フィルタ：<br/>AJ1-AJ7 のいずれかに違反するか?}
    Filter -->|違反する| Reject[（チーム）👆却下／哲学を再確認]
    Filter -->|違反しない| Build[（チーム）👆実装]
    Reject --> Doc[（チーム）📝アンチジョブ違反ログを残す]
    Doc --> Loop[（チーム）❤️境界を守る経験が文化として蓄積]

    classDef rejected fill:#f8d7da,stroke:#721c24,color:#000000;
    classDef accepted fill:#d4edda,stroke:#155724,color:#000000;
    classDef culture fill:#fff3cd,stroke:#856404,color:#000000;

    class Reject,Doc rejected;
    class Build accepted;
    class Loop culture;
```

> **メタ的な意味**：これはユーザージャーニーではなく、**プロダクトチームのジャーニー**。CQP は内部の意思決定プロセスにアンチジョブを組み込まないと、外向きの境界を保てない。

---

## このバージョンを採用するときに変わること

- 機能 backlog に「**境界フィルタ**」が入る
- プロダクト・ロードマップに「**意図的に作らないリスト**」が公開される
- マーケティングが「**ない機能の自慢**」になる
  - 「ランキングはありません」
  - 「課金はありません」
  - 「学習進捗トラッカーはありません」
  - 「親への監視機能はありません」
- ユーザーが選ぶ理由が「機能の多さ」ではなく「**哲学の一致**」になる
- 採用基準に「アンチジョブを尊重できる人」が入る

## 既存ジャーニーへのインパクト

| 既存ジャーニー | アンチジョブによる強化点 |
|---|---|
| CJ20（演出 ON/OFF 体験） | AJ1（学習目的化しない）：演出の良さを「教える」ではなく「**体感させる**」 |
| CJ22（フィードバック反映） | AJ4（コミュニティ大規模化しない）：友達は数人で十分 |
| CJ31-CJ34（承認） | AJ6（AI 判断者化しない）：判断は常に子 |
| CJ26（自分たちのゲーム） | AJ7（親リソース編集しない）：見た目・音は親子の手で |
| CJ43（実公開ログ） | AJ5（親管理過剰化しない）：観察用、監視用ではない |
| CJ35-CJ41（ガードレール） | AJ6（AI 判断者化しない）：壊れた判断はビルドが、創造の判断は子が |
