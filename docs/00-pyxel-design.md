# Pyxel版 ブロッククエスト 技術設計

元のHTML5 Canvas版ドキュメント群をPyxelの制約に読み替えた設計書。

---

## 1. 基本仕様

| 項目 | 元（HTML5版） | Pyxel版 |
|------|-------------|---------|
| エンジン | HTML5 Canvas 2D | Pyxel (Python) |
| 解像度 | 640x480 (20x15タイル, 32px) | **256x256** |
| タイルサイズ | 32px | **16px** (16x16タイル表示) |
| カラー | 任意色（ゾーン別多数） | **16色パレット**（カスタマイズ可能） |
| ファイル構成 | 単一 index.html | Python + .pyxres |
| セーブ | localStorage | **JSONファイル I/O** |
| 入力 | キーボード + 仮想ゲームパッド | **キーボード + ゲームパッド** |
| 音楽 | Web Audio API + TinyMusic (11曲) | **Pyxel内蔵サウンド 4ch** |
| 効果音 | OscillatorNode (20種) | **Pyxel内蔵サウンド** |
| 日本語 | Canvas fillText | **BDFRenderer + umplus BDFフォント** |

## 2. 画面構成

### 解像度 256x256、タイルサイズ 16px

- 表示可能タイル: 16x16 (ただしステータスバー分を差し引く)
- ステータスバー: 画面上部 24px (1.5タイル分)
- 実マップ表示: 16x14.5タイル → 実質 **16x14タイル** (256x224px)
- メッセージウィンドウ: 画面下部に重ねて表示

### 画面レイアウト

```
┌──────────── 256px ────────────┐
│ [ステータスバー] HP MP Lv     │ 24px
├───────────────────────────────┤
│                               │
│         マップ表示            │ 224px
│        (16x14タイル)          │
│                               │
│  ┌─────────────────────────┐  │
│  │ [メッセージウィンドウ]  │  │ ← 必要時のみ重ねて表示
│  └─────────────────────────┘  │
└───────────────────────────────┘
```

## 5. 描画

### Pyxelで使えないもの（やらない）

- アルファ透過（rgba）→ 使わない
- グラデーション → 使わない
- shadowBlur / グロー → 使わない
- globalCompositeOperation → 使わない
- clipRect → 使わない

### タイル描画方式

.pyxresのイメージバンクにタイルを描き、`pyxel.blt()` で描画。

- イメージバンク0: タイルセット (16x16px × 各種)
- イメージバンク1: キャラクター/敵スプライト
- イメージバンク2: 日本語フォント用 (BDFRenderer)

### タイルバリエーション

座標ハッシュで2-3バリエーションを選択（元仕様の考え方を踏襲）。
バリエーションはイメージバンク内に並べて格納。

### アニメーション

- 水タイル: 2フレームのパターン切替（`pyxel.frame_count` で制御）
- ヒーロー歩行: 2フレームの足切替
- 城の旗: 2フレーム
- 町の煙: 省略（16色+小解像度では視認困難）
- 洞窟の闇の明滅: 省略

### バトル画面

- 上半分: ゾーン別に単色背景（グラデーションなし）
- 下半分: 地面色（単色）
- 敵スプライト: イメージバンク1から `blt()` で描画
- 被ダメージ: スプライトのシェイク（座標オフセット）のみ
- 白フラッシュ/ワイプ演出: 省略

## 6. 日本語テキスト

### 方式

Pyxel公式サンプル `13_bitmap_font.py` の `BDFRenderer` クラスを利用。
BDFフォントファイル `umplus_j10r.bdf` を使用（10ドット、漢字含む7,187文字）。

### テキスト表示仕様

- フォントサイズ: 10px（umplus_j10r）
- 1行の最大文字数: 約25文字（256px幅 ÷ 10px）
- メッセージウィンドウ: 最大3行表示
- 自動改行: 文字数で折り返し

### メッセージシステム

```python
class MessageSystem:
    queue: list[str]     # メッセージキュー
    current: str | None  # 現在表示中
    lines: list[str]     # 改行済みの行リスト
    
    def show(self, text):
        """テキストをキューに追加"""
    
    def next(self):
        """Enterで次のメッセージへ"""
    
    def draw(self):
        """メッセージウィンドウを描画"""
```

### テキスト量について

元ドキュメントのテキストはそのまま使用。
Pyxelの小画面に合わせて自動改行するが、内容は削らない。

## 8. セーブ/ロード

### 方式

JSON形式でファイルに保存。

```python
import json

SAVE_FILE = "save.json"

def save_game():
    data = {
        "save_version": 1,
        "x": player.x, "y": player.y,
        "hp": player.hp, "max_hp": player.max_hp,
        # ... 全プレイヤーステータス
    }
    with open(SAVE_FILE, "w") as f:
        json.dump(data, f)

def load_game():
    try:
        with open(SAVE_FILE, "r") as f:
            data = json.load(f)
        # playerに復元
        return True
    except FileNotFoundError:
        return False
```

## 14. 省略・変更点まとめ

### 完全に省略するもの

| 元仕様の項目 | 理由 |
|-------------|------|
| 仮想ゲームパッド | Pyxelはデスクトップアプリ |
| レスポンシブレイアウト | 固定256x256 |
| CSS変数/DOM操作 | Pyxelには不要 |
| タイルキャッシュ（OffscreenCanvas） | Pyxelの`blt()`が十分高速 |
| アルファ透過エフェクト | Pyxelでは不可能 |
| グラデーション背景 | Pyxelでは不可能 |
| グロー/シャドウ | Pyxelでは不可能 |
| globalCompositeOperation | Pyxelでは不可能 |
| clipRectワイプ演出 | Pyxelでは不可能 |
| 縦持ち/横持ち対応 | 固定画面 |
| Node.js テスト統合 | pytest等で代替 |

### 簡略化するもの

| 元仕様の項目 | Pyxel版 |
|-------------|---------|
| ゾーン別パレット(多色) | 16色パレット1種 + タイル密度で表現 |
| 11曲BGM | 8 Music枠に収める（一部兼用） |
| 町の煙アニメ | 省略 |
| 洞窟闇明滅 | 省略 |
| 白フラッシュ演出 | 色反転(`pyxel.pal`)で代替検討 |
| 被ダメージワイプ | シェイクのみ |
| タイトルロゴ浮遊演出 | 静的表示またはシンプルな点滅 |

### 読み替えるもの

| 元仕様 | Pyxel版 |
|--------|---------|
| Canvas 2D API | `pyxel.blt()`, `pyxel.rect()`, `pyxel.text()` |
| Web Audio API | `pyxel.sound()`, `pyxel.music()`, `pyxel.play()` |
| localStorage | JSON ファイル I/O |
| requestAnimationFrame | `pyxel.run(update, draw)` |
| 絵文字（🛠🎉） | テキストで代替（「*DEBUG*」「!!」等） |
| navigator.maxTouchPoints | 不要（常にキーボード/ゲームパッド） |
