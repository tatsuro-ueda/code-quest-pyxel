# 実験案 v10：顧客のジョブ ── v1 × v2 ハイブリッド（競合の欠落 × 人格遷移）

## 1. 要約

### ある日のシーン：「スライムが強すぎ！」

君、RPG を改造して遊んでる。スライムが強すぎて勝てない。

**🅰️ Scratch だったら？**
「自分で直すしかない」。お父さんは横で見てるだけ。**君ひとりの戦い**になる。

**🅱️ Roblox だったら？**
Lua っていう難しい言語で書かないと直せない。お父さんも書けない。**諦める**。

**🅲️ お父さんがコードを直接書き換えるなら？**
お父さんが 30 分かけて直す。でも待ってる間に君はゲームから離れちゃう。「直したよ」って言われた頃には、**もう熱が冷めてる**。

**🆎 CQP だったら？**

3〜5 分の間にこんな感じで進む：

1. 君「スライム強すぎ！」👈 **遊んでる気持ち**
2. お父さん「AI さん、HP を 50→30 にして」👈 お父さんは **「君の言葉を AI に通訳する係」**
3. 機械「壊れてないかチェック…OK」👈 お父さんの **「番人モード」**（壊れたバージョンが君の画面に届かないように見張ってる）
4. 君、弱いバージョンと元のバージョンを両方遊んで比べる 👈 **判定する気持ち**
5. 君「弱いほうがいい！こっち！」👈 **自分で決めた気持ち**
6. また続きを楽しむ 👈 **遊んでる気持ちに戻る**

ここで大事なのは、**気持ちが 6 回バトンタッチしてる**ってこと。バトンが一度も床に落ちない。

## 2. 人格別ジョブ

### 子(プレイヤー)

> **JP1**: 自分が改造したゲームを、操作のストレスなく遊びたい  
> **JP2**: 困ったとき（敵強すぎ／詰む／迷う）に「直して！」と言える状態でありたい  
> **JP3**: 冒険を最後までやり切って、自分たちの作品を体験として完結させたい

**競合の位置**（JP1：操作のストレスなく遊ぶ）

| 競合 | 立ち位置 |
|---|---|
| Scratch | 完成度が低く没頭しにくい |
| Roblox | 80% 未完成で「自作を遊ぶ」段階に届かない |
| SMM2 | 自作ステージは遊べるが「自分の RPG」感が出ない |
| vibe coding 単体 | 壊れた版で集中が中断される |
| **CQP** | ヘッドレス検証＋承認キューで、常に動く版だけが届く |

---

### 子(クリエイター)

> **JCR1**: 英語や文法に翻弄されずに、視覚／聴覚で「直接」素材を触りたい  
> **JCR2**: 自分の作業の結果が、すぐに動いて見えてほしい  
> **JCR3**: 親に代筆されたくない（「自分で描いた」「自分で並べた」と言える）

**競合の位置**（JCR1+JCR3：直接触る × 代筆されない）

| 競合 | 立ち位置 |
|---|---|
| Scratch | ブロックは触れるが、見た目はスクラッチ感が抜けない |
| Roblox | Lua の壁で「自分で作った」感が遠い |
| SMM2 | 配置はできるが見た目はマリオ固定 |
| RPG Maker | 深いが時間がかかり、子は親の作業を見る側になりがち |
| vibe coding 単体 | AI に任せると「自分で作った」感が消える |
| **CQP** | リソースエディター（Tilemap/Sprite/Music/Sound）で直接触る／AI は build/verify のみ |

---

### 子(批評家)

> **JCT1**: 採否を、親の理屈ではなく**自分の体感**で決めたい

**競合の位置**（JCT1：体感で採否を決める）

| 競合 | 立ち位置 |
|---|---|
| Scratch | 親が関与しないので「採否を判断する経験」自体が無い |
| Roblox | 「いいね」中心で評価が他人軸に逃げる |
| RPG Maker | 親が直接編集 → 子は完成品を渡される側 |
| vibe coding 単体 | AI のおすすめを選ぶ構造になりがち |
| **CQP** | 「開発版／本番」の遊び比べで、自分の体感を根拠に judge する（独自） |

---

### 子(社会人)

> **JS1**: 友達がインストールも登録もせずに、その場で遊んでくれる状態にしたい  
> **JS2**: 友達の反応を、自分のサイクルにすぐ持ち帰りたい  
> **JS3**: 「遊ばれた／遊ばれなかった」を、勘ではなく事実で知りたい

**競合の位置**（JS1：友達が即プレイ）

| 競合 | 立ち位置 |
|---|---|
| Scratch | PC ブラウザ前提で、スマホでの友達体験が弱い |
| Roblox | アプリインストール／課金圧／家族の枠を超えた社会圧 |
| SMM2 | Switch + Nintendo Online が必要 |
| 自前配信ゲーム | スマホで動かない／.exe 怖い |
| **CQP** | URL 一発でスマホ即プレイ／実公開アクセスログで届きを事実化 |

---

### 親(養育者) ── 親という立場から来る根源ジョブ

> **JPR1**: 子どもと**コミュニケーションを取りたい**（一緒に過ごす時間を持ちたい・先生／生徒ではなく仲間として関わりたい）
> **JPR2**: 子どもに**合った成長の機会を与えたい**（年齢・関心・性格に合わせて、押し付けず・取り上げず）
> **JPR3**: 子どもの**主体性が育つ場を提供したい**（自分で決められる経験を積ませたい）
> **JPR4**: 親としての**手応えを感じたい**（「あの夏に一緒に作った」と振り返れる関係でいたい）

> **JPR1-JPR4 が「親が CQP を使う根源的な動機」**。続く 3 役（翻訳者・セーフティネット・観察者）は、これを果たすための戦術的手段。

**競合の位置**（JPR1：コミュニケーション × JPR3：主体性の場）

| 競合 | 立ち位置 |
|---|---|
| Scratch | 親が関わる経路が薄い／親は応援者止まり |
| Roblox | 関わるなら Lua、関わらないなら 80% 未完成（JPR1 と JPR2 がトレードオフ） |
| SMM2 | 並んで遊べるが「合った成長機会」までは設計されていない |
| RPG Maker | 親が直接編集 → 子は受け手（JPR3 が侵害） |
| vibe coding 単体 | 親が AI と直接話す → 子は結果を渡される側（対話ではなく報告） |
| 教育系 SaaS | 「学習進捗」主軸（JPR2 が「学習成果」に矮小化） |
| **CQP** | 翻訳者・セーフティネット・観察者の 3 役で JPR1-JPR4 を機構化 |

---

### 親(翻訳者)

> **JTR1**: 子の言葉（「強すぎ」「つまんない」「もっとカッコいい」）を、AI が動ける指示文に翻訳する

> **JPR1（コミュニケーション）と JPR3（主体性の場）を直接果たす役**。子の言葉が AI に届く経路を作ることで、親子の対話と子の主体的な要望表明が両立する。

**競合の位置**（JTR1：子の言葉を AI に翻訳）

| 競合 | 立ち位置 |
|---|---|
| Scratch | 翻訳ジョブが存在しない（子が直接やる） |
| Roblox | 親が Lua を書ける必要 → 親が「先生」化する |
| RPG Maker | 親が編集者か、子独力かの二択 |
| vibe coding 単体 | 親が直接 AI に話す → 翻訳ではなく代弁になる |
| **CQP** | 承認キューが「親＝代弁者」経路を塞ぎ、翻訳者役に純化させる |

---

### 親(セーフティネット)

> **JSN1**: 壊れた版を子の画面に届けない  
> **JSN2**: 親自身の「しゃしゃり出る」衝動も止める

> **JPR2（合った成長の機会）を守る役**。壊れた版や、親の善意の暴走から子を守ることで、「成長機会」が「侵害」にならないようにする。

**競合の位置**（JSN1+JSN2：壊れた版／親の暴走を止める）

| 競合 | 立ち位置 |
|---|---|
| 全競合 | システムとしてのセーフティネット機構を持たない／親の自制に依存 |
| **CQP** | ヘッドレス検証・承認キュー・セーブ互換テストで機構化（独自） |

---

### 親(観察者)

> **JOB1**: 今日の子の関心がどこにあるか察する（マップ？戦闘？セリフ？演出？）／集中が切れる兆候を読む

> **JPR2（合った成長の機会）と JPR4（手応え）を支える役**。子の関心の所在を察することで、押し付けでも放置でもない「合った機会」を見つけられる。同時に「自分は子の今を見られている」という親としての手応えにも繋がる。

**競合の位置**（JOB1：子の関心と疲れを察する）

| 競合 | 立ち位置 |
|---|---|
| Scratch | 親が関わらないので観察対象が薄い |
| Roblox | 保護者管理機能はあるが**監視寄り**（観察と監視の混同） |
| 教育系 SaaS（CodaKid 等） | 親レポートが詳細すぎて監視化／プライバシーの侵食 |
| **CQP** | 最小限の指標（最終プレイ日時、修正サイクル時間、関心の遷移）で「観察」に留める |

---

> **JF3**: 率直に「ここ難しい」「ここ楽しい」と言える距離感でいたい
