# ストーリーコンセプト

## テーマ（一行定義）

「見えてしまう者は、世界の歪みを無視できない」

## コアメッセージ

役割は与えられるものじゃない。
「見えてしまう人」にしか担えない仕事がある。
好きは、責任の入口でもある。

## PREP構造

### Point（結論）
ストーリーの軸は「役割は押しつけではなく、"見えてしまう人"が引き受けるもの」。
これをNPCのセリフ変化、ゾーン進行の意味付け、魔王との対話で一貫させる。

### Reason（なぜこの設計が強いか）
- 「プログラミング好き＝能力」ではなく「世界の違和感に気づいてしまう性質」に再定義できる
- RPG構造（成長・敵・ボス）と自然に一致する
  - 子ども：使命の物語として読める
  - 大人：生きづらさの再解釈として読める

### Example（具体構造）

#### 主人公の内面変化アーク
```
違和感 → 反発 → 孤独 → 自覚 → 選択
```

各ゾーンがこのアークの1フェーズに対応する。

#### メタファー再定義

| RPG要素 | 従来の意味 | 本作での意味 |
|---------|-----------|-------------|
| 敵 | 障害 | 思い込みの結果（暴走したブロック） |
| 戦闘 | 攻撃 | 観察・整理 |
| 攻撃 | 力で倒す | 順番を見直す・規則を見つける |
| 回復 | 体力回復 | 安心・落ち着きを取り戻す |
| ダメージ | 体力減少 | 思考の乱れ・集中の崩壊 |
| 状態異常（バグ汚染） | 毒 | 思い込みが入り込む |
| 勝利 | 討伐 | 理解・役割の受容 |
| 成長（レベル） | 強くなる | 理解が深まる |
| スキル習得 | 能力獲得 | 新しい「見方」を手に入れる |
| 敗北 | 死亡 | まだ整理できていない（自己否定にならない） |
| 最終ボス | 倒すべき悪 | 同じ性質を持った者が堕ちた姿 |

### Point（まとめ）
プレイヤー体験の変換：
- 戦闘 → 観察
- 攻撃 → 整理
- 回復 → 安心
- 勝利 → 自己理解

「ゲームをプレイしている」→「自分を理解している」

## ゾーンごとのテーマ設計

各ゾーンは主人公の内面変化の1フェーズに対応する。

### Zone 0：違和感（逐次処理）
- 「順番って大事じゃない？」という違和感
- 町の人は「気にしなくていい」側。プレイヤーは疎外感を覚える
- 👉 主人公の特性の芽

### Zone 1：反発（ループ）
- 同じことを繰り返す世界への疑問
- 「なんで気づかないんだ？」が生まれる
- 👉 初めて"他人とのズレ"を自覚

### Zone 2：理解の孤独（条件分岐）
- 「場合によって変わるのに…」と気づく
- でも説明しても伝わらない
- 👉 孤独フェーズ

### Zone 3：役割の自覚（変数・クローン）
- 世界は「数」と「状態」でできていると理解する
- 自分だけがそれを見えている
- 👉 逃げるか、引き受けるかの分岐

### Zone 4：選択（グリッチ）
- 同じ性質を持った者が堕ちた姿と対峙する
- 👉 「役割」を選ぶ

---

## ボスの正体（セリフで表現する設計）

魔王グリッチは「主人公と同じ性質を持ちながら、理解されなかった者」。
セリフだけで表現する：

> グリッチ「昔は…お前と同じだった」
> グリッチ「でも誰も理解しなかった」
> グリッチ「だから全部壊すことにした」

追加設定：
- 純粋な少女だった
- 純粋さは「無垢で信じやすい」「正しさをまっすぐ求める」「承認を求めてしまう」を併せ持つ
- プロフェッサーに娘のように育てられた
- プレイヤーへの感情の中心は「理解されたい願い」

グリッチのセリフだけのパート案：

> グリッチ「わたしは、まちがえたかったわけじゃない！」
> グリッチ「ちゃんとできれば、ほめてもらえると思ってた」
> グリッチ「いい子でいれば、ここにいていいって言ってもらえると思ってた！」
>
> グリッチ「パパは、むずかしいことをたくさん教えてくれた」
> グリッチ「わからないままでも、うなずけば笑ってくれた！」
> グリッチ「だから、もっとがんばろうって思った」
>
> グリッチ「こわせって言われたわけじゃない」
> グリッチ「ただ、まもろうとしただけ！」
> グリッチ「なくしたくなかっただけ」
>
> グリッチ「でも、まもりかたがわからなかった！」
> グリッチ「こわしたら、もう離れていかない気がした」
> グリッチ「こわしてしまえば、ずっと見てもらえる気がした…」
>
> グリッチ「おまえは、見るんだな？」
> グリッチ「バグじゃなくて、わたしを」
> グリッチ「……それなら、少しだけ、こわい！」
>
> グリッチ「わたしのことを、わかってほしい」
> グリッチ「まちがっていたことも！」
> グリッチ「さびしかったことも」
> グリッチ「それでも、ここにいたかったことも……！」

構造：通信塔の声（「考えるな」「みんなと同じでいい」）を信じた未来＝グリッチ

撃破後、グリッチは**消えない**。理解は排除ではない。

## ランドマークの象徴的意味

| 要素 | 世界樹 | 通信塔 |
|------|--------|--------|
| 役割 | 内側 | 外側 |
| 感情 | 安心 | 圧迫 |
| メッセージ | 考えていい | 考えるな |
| 影響 | 自己肯定 | 思い込み |

### 世界樹（Zone 0）＝ 内側の肯定
> 世界樹だ
> なぜか落ち着く
> 気持ちが、考えが、自由だ

「考えたい衝動」は異常ではない。安心できる場所。

### 通信塔（Zone 2）＝ 外側の圧力
> 通信塔だ
> 声が流れてくる
> 「そんなに考えなくていい」
> 「みんなと同じでいい」

社会の圧力。思い込みの発生源。

### ボス戦との接続
プレイヤーはすでに体験している：
- 世界樹 → 「考えていい」
- 通信塔 → 「考えるな」

グリッチ戦の意味：通信塔の声を信じた未来＝グリッチ

グリッチのセリフ：
> 「その声を聞き続けた結果が、これだ」

## UI全体の哲学

ストーリーはイベントシーンだけで語るものではない。
プレイヤーが最も多く触れるUIテキスト（戦闘ログ、ショップ、セーブ）が哲学を担う。

### ショップ・宿屋の言葉
- 宿屋＝「思考を整理する」場所。データ復元ではなく、息をつく行為
- 武器購入＝「整理する手段が増えた」
- 防具購入＝「思考が少し安定する」
- セーブ＝「ここまでの理解を書き留めた」
- ロード＝「記録を読み返した。理解が戻ってくる」

すべてのテキストが「理解の旅」の一部であること。

### 戦闘ログの哲学

### なぜ戦闘ログが最重要か
- プレイヤーが最も多く読むテキスト
- ここが浅いと全体が浅くなる
- ここが深いとストーリーを説明しなくても伝わる
- 👉 UIが"哲学"になるポイント

### 設計原則
- 「攻撃の説明」→「理解プロセスの言語化」に変える
- プレイヤーは「戦っている」のではなく「気づいている」感覚にする
- 敗北を自己否定にしない
- 逃走も「目をそらす」「一度離れて考え直す」という表現にする
