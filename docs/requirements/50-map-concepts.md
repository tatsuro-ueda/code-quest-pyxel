# マップ設計思想

**ブロッククエスト 〜コードの王国を救え〜**

## 核心原則

> **「絵から作るのではなく、体験から作る」**

マップは見た目ではなく、プレイヤーの体験を第一に設計する。
美しいビジュアルは、体験設計の後から乗せる装飾である。

---

## マップ作成の7ステップアプローチ

### ステップ1: 一本道を作る（骨格）

**目的**: 初心者でも迷わない基本動線の確立

- スタート地点からゴール地点まで、一直線の道を作る
- これが全体の「骨格」となる
- プレイヤーが迷う要素をゼロにする
- シンプルだが、確実にゴールへ導く

```
Start ━━━━━━━━━━━━━━━━━━━> Goal
```

### ステップ2: 分岐を1つだけ足す

**目的**: 探索欲求の創出

- メインルートから横道を1つだけ追加
- 宝箱や隠しアイテムを配置
- 「寄り道したい」という気持ちを生む
- まだ迷うほどではない

```
Start ━━━━━━━┳━━━━━━━━━━> Goal
              ┃
              ┗━> 宝箱
```

### ステップ3: ランドマークを置く（目印）

**目的**: 方向感覚の喪失を防ぐ

- 大きな樹、塔、岩などの視覚的目印を配置
- 遠くからでも見える特徴的なオブジェクト
- 「あの木のところに戻ればいい」という安心感
- プレイヤーの脳内地図形成を助ける

**具体例**:
- 大樹（3x3タイル以上）
- 塔や灯台
- 巨大な岩
- 特徴的な建物

### ステップ4: 高低差で進行を制御する

**目的**: 無言のルート誘導

- 崖、川、山などの自然な障害物を配置
- プレイヤーを強制せずに、自然に正しい方向へ導く
- 「行けそうで行けない」領域を作る
- 後で橋や階段で開通させることで進行感を演出

**手法**:
- 崖で遮る → 後で迂回路を発見
- 川で分断 → 後で橋を見つける
- 高台に宝 → 別ルートから回り込む

### ステップ5: 密度のメリハリをつける

**目的**: 視覚的注目の誘導

- 重要な場所ほど詳細に作り込む
- 通過点はシンプルに
- プレイヤーの注意を適切に配分

**密度の段階**:
- **高密度**: ボス前、村の中心、秘密の場所
  - 装飾オブジェクト多数
  - 多様なタイル種類
  - アニメーション要素
- **中密度**: 通常の探索エリア
  - 適度な装飾
  - 基本的なバリエーション
- **低密度**: 移動専用の道
  - 最小限の装飾
  - シンプルな構成

### ステップ6: シズル感を乗せる（光・影・アニメーション）

**目的**: 世界に生命感を与える

既に実装済みのシズルルールを適用:
- 水面のキラキラ（specular highlights）
- 草原の揺れ
- 炎の揺らぎ
- 影のグラデーション
- 時間帯の光の変化

**注意**: これは最後のステップ。体験設計が完了してから適用する。

### ステップ7: 「触れそう」を配置する

**目的**: プレイヤーエージェンシーの付与

- インタラクティブな要素を散りばめる
- 「これは何だろう？」という好奇心を刺激
- プレイヤーの能動的な行動を引き出す

**配置するもの**:
- 宝箱（開けられる）
- 看板（読める）
- 草むら（切れる）
- タル・壺（壊せる）
- NPC（話せる）
- 扉（開けられる）

**配置原則**:
- 視界に常に1-2個は存在する密度
- ランダムではなく、意図的な配置
- リワードは期待に見合うものを

## 現状の問題点

### 技術的制約

1. **DQ1スタイルの単一セルタイル**
   - 1マスに1種類の地形のみ
   - 遷移が不自然（草→水が直角）
   - モノトーンで単調

2. **ランダム地形配置**
   - Perlinノイズによる自動生成
   - 意図のない配置
   - 体験設計がない

3. **同心円ゾーンシステム**
   - 中心から外側に向かって単調に変化
   - バイオーム境界が不自然
   - 探索の驚きがない

4. **自然な障壁の欠如**
   - すべての場所に行けてしまう
   - 進行制御ができない
   - ルート誘導がない

5. **一様な密度**
   - すべての場所が同じ詳細度
   - メリハリがない
   - 重要な場所が目立たない

## 目指すクオリティ

### 参考: 聖剣伝説2（Secret of Mana）レベルの詳細度

**具体的な要素**:

#### 1. オートタイル遷移
- 草地と水の境界に自動的に砂浜を生成
- 滑らかな地形遷移
- 角の処理（内角・外角）
- 多様な接続パターン

#### 2. 装飾レイヤー
- **花**: ランダムだが意図的な配置
- **石**: 道沿いや崖際
- **キノコ**: 森の中
- **小草**: 草原のアクセント

#### 3. 有機的な曲線道
- 直線的すぎない自然な道
- 地形に沿った曲がり方
- 幅の変化

#### 4. マルチタイルランドマーク
- 2x2, 3x3の大型オブジェクト
- 複数タイルで構成される建物
- 視認性の高い目印

#### 5. ゾーンベースの地形キャラクター
- 森エリア: 緑豊か、木が多い、薄暗い
- 砂漠エリア: 乾燥、岩、サボテン
- 雪原エリア: 白、氷、針葉樹
- 各ゾーンに明確な個性

## 制約事項（やらないこと）

### 技術的に実装しない機能

1. **フルスプライトシート**
   - 複雑なタイルセット管理はしない
   - プロシージャル生成を維持

2. **パララックススクロール式オーバーワールド**
   - 複数レイヤーの背景スクロールなし
   - シンプルな2D表現に留める

3. **マルチレイヤータイルマップエディタ**
   - Tiledのような複雑なエディタは使わない
   - コード内での生成を維持

### 開発方針

**「単一HTML内でのプロシージャル生成」にこだわる**

- すべてJavaScriptで生成
- 外部アセット最小限
- アルゴリズムによる美しさ
- コードで体験を記述

## まとめ

マップは**プレイヤーの体験を設計するツール**である。

1. まず体験（迷わない、探索したい、発見がある）を設計
2. 次に構造（骨格、分岐、障壁）を作る
3. 最後にビジュアル（密度、シズル、装飾）を乗せる

この順序を守ることで、美しいだけでなく**遊んで楽しいマップ**が生まれる。

技術的制約の中で、最大限の体験価値を提供する。
それが「ブロッククエスト」のマップ設計哲学である。
