# 実験案 v8：顧客のジョブ ── 教育哲学軸（Constructionism / SDT / PBL）

> 実験ラベル：**v8 / 教育哲学軸**
> 作成日：2026-04-25
> 視点：CQP のジョブを **既存の教育哲学のフレームワーク** で再構成する。Seymour Papert の Constructionism、Deci & Ryan の自己決定理論（SDT）、Project-Based Learning（PBL）、モンテッソーリの「敏感期」概念。これらの**先行文献の言語**でジョブを書き直すことで、プロダクトの**理論的正統性**と**何を**主張**しているか**が鮮明になる。
> 動機：v1-v7 はいずれも独自のフレームワークだった。v8 は**他者の標準言語**で書く。教育者・研究者・投資家との対話で使える共通語彙を持つことの戦略的価値。

## 0. 4 つの教育哲学とその核心

### 0.1 Constructionism（構成主義、Seymour Papert, 1980）

> 「子どもがコンピュータをプログラムする」のであって、「コンピュータが子どもを教える」のではない（"the child programs the computer, not the computer programs the child"）。

学習は**何かを作る行為**を通じて起きる。「powerful ideas」（科学・数学・芸術の深いアイデア）に親密に触れる経路としての「ものづくり」。

### 0.2 自己決定理論（Self-Determination Theory, Deci & Ryan, 1985）

> 内発的動機を支える 3 つの基本的心理欲求：
> - **Autonomy（自律性）**：自分で決めている感覚
> - **Competence（有能感）**：自分はできるという感覚
> - **Relatedness（関係性）**：大切な人と繋がっている感覚

外発的動機（評価・報酬）はこの 3 つを侵食する。

### 0.3 Project-Based Learning（PBL）

> 学習は**真正性のあるプロジェクト**（authentic project）を通じて起きる。  
> 評価は最終産物と過程の両方。  
> 学習者は能動的に問題設定・問題解決をする。

### 0.4 モンテッソーリの「敏感期」

> 子どもには発達段階に応じた**敏感期**があり、特定の学習対象に強い興味を示す。  
> 大人の役割は「教える」のではなく「**準備された環境**」を提供すること。

## 1. CQP のジョブを 4 哲学で再構成

### Constructionism × CQP

| Papert の主張 | CQP のジョブ翻訳 |
|---|---|
| 「子がコンピュータをプログラムする」 | **CJ-P1**: 子がリソース（タイル、スプライト、音）を直接操作して世界を作る |
| 「powerful ideas に触れる」 | **CJ-P2**: 「壁との衝突」「経験値カーブ」「分岐」など**ゲーム設計の本質的アイデア**に親密に触れる |
| 「ものづくりが学びの形」 | **CJ-P3**: 完成品（自分たちのゲーム）が学びの証拠。テストではなく作品。 |
| 「教師は媒介者（mediator）」 | **CJ-P4**: 親（教師）は AI に翻訳する媒介者。「正解を教える」のではない。 |

**設計に降ろすと**：
- リソースエディター（CJ23, CJ24）は Papert の Logo の精神的後継
- 親(翻訳者) のジョブは**メディエーター・ロール**として位置づけ直される
- 「学習進捗 UI を作らない」（v6 AJ1）はまさに Papert の "instructionism 反対" を具現化

---

### Self-Determination Theory × CQP

| SDT の心理欲求 | CQP のジョブ翻訳 |
|---|---|
| **Autonomy（自律性）** | **CJ-S1**: 子が「採否を体感で決める」承認キュー（CJ31-CJ34） |
| **Competence（有能感）** | **CJ-S2**: 「直った！見て！」の達成感（CJ01, CJ08）／作品の完成（CJ30, CJ42） |
| **Relatedness（関係性）** | **CJ-S3**: 親子の対等関係（Job1）／友達フィードバック（CJ22）／家族 identity（v5 L4） |

**設計に降ろすと**：
- 承認キューは**Autonomy** の機構
- 数分で 1 周のループは**Competence** の連続経験
- 親(翻訳者)・友達フィードバック・実公開ログは**Relatedness** のインフラ

**重要な含意**：v6 のアンチジョブ（評価しない、ランキングしない、課金しない）は、**SDT の文脈で言えば「外発的動機の侵入を防ぐ境界」**。SDT の理論的根拠を持つ。

---

### Project-Based Learning × CQP

| PBL の要件 | CQP のジョブ翻訳 |
|---|---|
| **真正性のあるプロジェクト** | **CJ-PB1**: 「自分たちが遊ぶ／友達に見せる」現実のゴール |
| **学習者主導** | **CJ-PB2**: 子が要望を出し、判断を下す（承認キュー、フィードバック反映） |
| **協働的探究** | **CJ-PB3**: 親子・友達との協働（CJ25, CJ22, CJ26） |
| **公開された成果物** | **CJ-PB4**: スマホ URL で友達に届ける（CJ21, CJ43） |
| **過程と産物の両方の評価** | **CJ-PB5**: 履歴 UI で進化の跡を残す（v3 RJ3.3 と整合） |

**設計に降ろすと**：
- CQP は構造的に**完璧な PBL プロダクト**。教育プロダクトとして PBL 文脈で説明可能。
- 「評価しない」（AJ2）と PBL の「過程と産物の評価」は対立しない——プロダクトは評価しない／**家族と友達の自然な反応**が評価機能を果たす。

---

### モンテッソーリ × CQP

| モンテッソーリの概念 | CQP のジョブ翻訳 |
|---|---|
| **準備された環境（prepared environment）** | **CJ-M1**: 既存の Block Quest が「触って改造できる出発点」として準備されている |
| **敏感期** | **CJ-M2**: 子の関心がマップ→敵→セリフ→演出と移ろうのを**プロダクト内で吸収**できる（v4 Phase 2） |
| **自己訂正（self-correction）** | **CJ-M3**: 開発版／本番の遊び比べが**自己訂正の機構**（教師が訂正しない） |
| **大人の役割は不介入の介入** | **CJ-M4**: 親(観察者) のジョブはまさにこれ |

**設計に降ろすと**：
- 起動時に「本番版」が selector に並ぶことが**準備された環境**
- 一覧表の分類（開発／デバッグ／演出／共有／発展）は**敏感期の選択肢**
- 承認キューの「体感差」は**自己訂正**の場
- 親(観察者) ダッシュボード（v2）はモンテッソーリ的観察を支援する

## 2. 4 哲学を統合した CQP のメタジョブ

### Meta-Job MJ1: "Powerful Ideas" を**ゲーム作りの中で**触れる場を提供する

> Papert の「powerful ideas」（深いアイデア）に、**抽象的な学習教材ではなく、具体的なゲーム改造**を通じて触れる。

具体例：
- 「壁との衝突」→ 当たり判定ロジック（CJ12）
- 「経験値カーブ」→ 数学的成長関数（CJ29）
- 「分岐」→ 条件論理（CJ27）
- 「フィードバックループ」→ 制御理論（CJ22）

これらは**学校の教科**で出会うより**ゲーム作りで先に出会う**。Papert の理想。

---

### Meta-Job MJ2: SDT の 3 欲求を**家族単位で**満たす

> 個人の Autonomy / Competence / Relatedness ではなく、**家族の**3 欲求を満たす。

- 家族の Autonomy：「私たち家族は自分たちで決めて作る」
- 家族の Competence：「私たち家族は何かを完成させられる」
- 家族の Relatedness：「私たち家族は外の世界（友達）と繋がっている」

これは個人主義的な SDT を**家族集団に拡張**する解釈。CQP の独自性。

---

### Meta-Job MJ3: 真正性のあるプロジェクトとしての**家族の作品**

> PBL の「真正性のあるプロジェクト」を、**学校の課題**ではなく**家族の作品**として再定義。

- ゴール：自分たちで遊ぶ／友達に見せる（CJ21, CJ42）
- 成果物：完成した RPG（CJ30）
- 評価：友達の対面反応（CJ22）／家族の振り返り（v3 RJ3.3）

---

### Meta-Job MJ4: 子の発達段階に応じた**準備された家庭環境**

> モンテッソーリ的「準備された環境」の家庭版。

- 起動 1 分で動く（CJ46 / Phase 0）
- 関心の遷移を吸収する一覧（v4 Phase 2 / 敏感期）
- 親が「教えない」介入の作法（親(観察者) のジョブ）

## 3. 4 哲学からの設計指針

| 哲学 | 設計に来る指針 |
|---|---|
| Constructionism | リソースエディターを「直接触る」インターフェースに保つ／親リソース直編集をしない（AJ7） |
| SDT (Autonomy) | 評価を画面に出さない（AJ2）／承認キューを最重要機構として保護 |
| SDT (Competence) | 1 周のサイクルを数分に保つ／達成感を継続供給 |
| SDT (Relatedness) | 親子の対等関係（Job1）／友達ループ（CJ22, CJ43） |
| PBL | 真正性のあるゴール（友達に届ける）／過程の履歴（履歴 UI） |
| モンテッソーリ | 準備された環境（起動 1 分）／敏感期の選択肢（一覧）／不介入の介入（観察者） |

## 4. このバージョンを採用するときに変わること

- プロダクトの**理論的正統性**が言語化される（教育者・研究者・投資家への説得力増）
- 教育系プロダクトとして PBL／Constructionism 文脈でブランディング可能
- 学校・教育機関への提案ストーリーが書ける
- 「先生 / 生徒化を回避」がモンテッソーリ・Papert の文脈で意味づけされる
- アンチジョブ群が SDT の「外発的動機の侵入防止」として理論的根拠を得る
- ジャーニーが教育研究の文脈で論文化可能（学術接続）

## 5. 教育哲学軸が Layer 4（v5）と整合する理由

| Layer 4 ジョブ | 対応する教育哲学 |
|---|---|
| 家族 identity の形成 | SDT の Relatedness（家族版） |
| 子の自己効力感 | SDT の Competence + Autonomy |
| 親子の対等関係 | Papert の "mediator" 思想 / モンテッソーリの不介入の介入 |
| 失敗の共有体験 | PBL の過程の評価 |
| 次の物語への接続 | モンテッソーリの敏感期の遷移 |

**含意**：v5（価値階層）と v8（教育哲学）は**同じ現象を異なる言語で記述している**。両方を持つことで、技術畑の人にも教育畑の人にも届く。

## 6. 教育哲学軸の戦略的含意

### 含意 1：教育系プロダクトとしての**ブランディング**

「Scratch の流れを汲む構成主義教育プロダクト」「家庭版 PBL プラットフォーム」と説明できる。

### 含意 2：教育機関との**パートナーシップ可能性**

学校との接続は AJ1（学習目的化しない）と矛盾するが、**家庭学習の補助**として親への支援パッケージを学校が推奨する形なら矛盾しない。

### 含意 3：研究・学会接続

教育工学・発達心理学・HCI 研究との接続が可能。論文・カンファレンス発表のテーマになる。

### 含意 4：教育系投資家・財団への訴求

教育系のインパクト投資家・財団は SDT・PBL・Constructionism の文脈に親しい。共通語彙でのコミュニケーション。
